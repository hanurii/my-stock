description: 점수 변동 사유 등 데이터 코멘트를 확인 없이 추론으로 작성하는 것은 절대 금지
type: feedback
---

원인을 정확히 모르면서 그럴듯한 이유를 지어내는 것은 절대 하지 않는다.

**Why:** 성장주 점수 변동 사유를 "기준금리 인하로 감점 완화"라고 자동 생성했으나, 실제 기준금리 인하 사실이 없었음. 부정확한 추론으로 사유를 조작한 것은 사용자에게 잘못된 판단 근거를 제공하는 심각한 문제.

**How to apply:**
- `grade_change_reason` 등 사유 필드는 확인된 사실만 기록한다
- 원인을 모르면 비워두고, 수치 변동만 표시한다
- 이전 데이터와 현재 데이터를 실제로 비교할 수 없는 상황에서 "아마 ~일 것이다"로 코멘트를 채우지 않는다
- feedback_report_data_verification.md의 원칙과 동일: 추론/환각 금지, 실제 조회+검증된 데이터만 사용
