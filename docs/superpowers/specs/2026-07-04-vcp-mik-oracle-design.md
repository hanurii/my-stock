# VCP 오라클 확장: 마이클스 컴퍼니(MIK) 2014-11-06 + 검출기 견고화 (설계 spec)

작성일 2026-07-04 · 상태: 구현 완료(feat/vcp-redesign), MIK as-of 2014-11-06 breakout 통과(coil-baseline OR-경로) · 대상 브랜치 feat/vcp-redesign

## 1. 배경·목적

미너비니가 책에서 실전 성공 사례로 제시한 마이클스 컴퍼니(The Michaels Companies,
NASDAQ: MIK, 2021년 상장폐지)가 2014-11-06 VCP 매수 포인트를 만족했다고 한다(책에는
차트 + 매수 포인트 날짜만 있음). 이 사례를 `vcp_examples.json`의 6번째 정답
예시(오라클)로 추가해 코일 피벗 검출기(2026-06-30 재설계,
`2026-06-30-vcp-final-coil-pivot-design.md`)를 검증하고, 어긋나면 진단이 시키는
만큼 수정한다.

현재 검출기 성적: 기존 5예시 중 인식 5/5, 돌파일 판정 3/5
(PASS: 데브시스터즈·한솔케미칼·다올2차 / 구조적 미충족: 켐트로스·다올1차 /
범위 밖: 메리츠).

## 2. 성공 기준

- MIK 돌파일(2014-11-06) as-of에서 `vcp_detected=True` + 피벗이 복원된 실제
  매수점 근사(기존 기준 ±2% 수준) + status=breakout(돌파 당일 기준).
- 회귀 가드(모두 유지):
  - 기존 PASS 3건(데브시스터즈 194480 · 한솔케미칼 014680 · 다올2차 030210) 유지.
  - 70종목 한국 유니버스 find-vcp 분포에 비합리적 변화 없음, 기가비스(420770)
    `vcp_detected=False` 유지.
  - 단위테스트 전부 통과.

## 3. 데이터 확보 (확인된 리스크)

MIK는 상폐 종목이라 통상 경로가 모두 막힘을 사전 확인했다(2026-07-04):
FDR/야후 404, stooq(.com/.pl) JS 봇 검증 차단, WSJ CSV 401. **봇 검증 자동 우회는
하지 않는다.** 확보 순서:

1. 웹 조사로 공개 미러(GitHub에 올라온 미국 일봉 아카이브, Kaggle 스냅샷 미러 등)에서
   mik.us 일봉 탐색.
2. 실패 시 사용자에게 브라우저 다운로드 요청(사람 브라우징은 정상 통과):
   `https://stooq.com/q/d/l/?s=mik.us&i=d&d1=20130601&d2=20150630`
3. 확보한 2013-06~2015-06 일봉을 `public/data/vcp_oracle_mik.json` 고정 파일로
   저장(키: dates/opens/highs/lows/closes/volumes — `load_series` 반환 형식과 동일).
   이후 네트워크 없이 영구 재현 가능(find-3c NU 검증과 같은 취지).
   저장 전 무결성 점검: 거래일 수(해당 구간 미국 영업일 근사), OHLC 관계
   (low≤open/close≤high), 2014-11-06 존재.

## 4. 책 명세 복원

책에 수치 명세가 없으므로 웹 조사(미너비니 책 발췌·강연·차트 해설)와 실데이터로
베이스 구간과 피벗 가격을 복원한다. 산출: 예시 항목의
`start`(베이스 시작) / `end`(돌파 직전 마지막 횡보일) / `breakout_date`(2014-11-06) /
`pivot`(매수점 가격). `note`에 복원 근거(출처·판단 로직)를 기록한다.
복원 결과가 2014-11-06과 모순되면(예: 실데이터상 그날 신고가 돌파가 없음) 진행을
멈추고 사용자에게 보고한다.

## 5. 컴포넌트 변경

### 5.1 예시 파일 스키마 확장 (`public/data/vcp_examples.json`)
- 항목에 선택 필드 `market`("US") · `data_file`(고정 데이터 파일 경로) 추가.
- 기존 한국 예시 5건(6항목)은 무변경 — 필드 없으면 기존 경로(FDR) 그대로.

### 5.2 로더 분기 (`scripts/canslim_lib/vcp_audit.py::load_series` 호출측)
- `data_file` 지정 시 FDR 대신 로컬 JSON 로드(start−버퍼~end 슬라이스는 동일 규칙).
- 실패 모드: 파일 없음/키 누락 → None 반환(기존 방어 규칙과 동일).

### 5.3 진단
- vcp-audit 5축 성적표(선행급등·수축·수축별 거래량·마름·돌파)를 MIK에 적용.
- 코일 검출기 as-of 판정(돌파일과 전일): vcp_detected/pivot/status/reason.
- 산출물: 축별 어긋남 리포트(어느 게이트·파라미터가 막는지).

### 5.4 조건부 로직 수정 (진단이 시키는 만큼만)
1. **파라미터 재보정**: 6예시 전체 + 70종목 가드 기준으로 코일 파라미터
   (coil_tight_pct/coil_min_days/coil_max_days/coil_dry_max) 재검.
2. **구조 수정(허용됨)**: 파라미터로 안 되면 탐지 구조 변경. 켐트로스·다올1차의
   미해결 구조 문제(횡보 거래량 안 마름)와 같은 뿌리로 판명되면 묶어서 해결을
   검토하되, 회귀 가드(§2)를 깨면서까지 통과시키지 않는다 — 그 경우 "구조적
   미충족"으로 기록하고 사용자 판단에 회부.

## 9. 결과 요약 (Task 1–5 완료)

- MIK as-of 2014-11-06: `vcp_detected=True`, `status=breakout`, 피벗(코일고가) 18.33 vs
  복원 저항/매수점 18.50(−0.9%, ≤2% 기준 충족). 원인 진단: MIK는 2014-06 IPO로 50일
  거래량선이 상장초기 고거래량에 부풀려져, 실제 돌파(종가 18.68, 저항 18.50 상향돌파)의
  거래량이 MA50×1.06(기존 게이트 1.4 미달 — "조용한 돌파")로만 보였으나 마른 코일
  평균거래량 대비로는 1.63×(뚜렷한 확장)였다.
- 수정(Option C, 사용자 승인): `_is_breakout`에 코일-기준선 OR-경로 추가 —
  `DEFAULT_PARAMS["coil_breakout_vol_mult"] = 1.5`. 거래량 확장 판정 =
  `vols[i] ≥ ma50×1.4` **또는** `vols[i] ≥ 코일평균거래량×1.5`. `coil=None`(기존 호출부)
  이면 (b) 경로 비활성 — 기존 동작 완전 보존. 이 OR-경로는 "dry-up 대비 확장(전체 적용)"이
  미너비니의 실제 규칙으로, 마른 코일이 존재하는 모든 종목에 일반 적용된다 — IPO로 MA50이
  오염된 MIK는 이 경로가 필요함을 드러낸 **동기 사례**일 뿐 적용 범위의 한계는 아니다.
  **모니터링**: 70종목 세트만 확인됨(breakout 1→3, 전부 기존 인식 종목의 상태 전환,
  vcp_count 16→16) — 다음 전체 유니버스 find-vcp 실행 시 breakout 건수 변화를 관찰할 것.
- 기존 3예시(데브시스터즈·한솔케미칼·다올2차) PASS 유지, 단위테스트 22/22 통과(신규 3건 포함).
- 70종목 회귀: `breakout` 1→3(+2, 신규 vcp_detected 증가 아님 — 기존 vcp_detected 종목의
  status만 정정), `vcp_count` 16→16(불변).
- **기가비스(420770) 참고 사항(본 작업 범위 밖)**: 이번 확인(2026-07-04) 시점에
  `vcp_detected=True`/`status=forming`으로 나타났다(직전 기록 시점엔 `False`). 가격
  캐시가 2026-06-30→07-03로 갱신되며 120일 lookback 창이 이동해 인식 게이트(net-수렴
  판정)의 통과 여부가 바뀐 **사전 존재 창-민감도** 문제로 추정 — 이번 작업(돌파 거래량
  게이트만 변경)과는 무관하며, 변경 전/후 동일하게 forming(거래신호 아님)이라 회귀는
  아니다. 사용자에게 플래그, MIK 오라클 범위 밖.
