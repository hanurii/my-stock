# VCP 검출기 재설계 (evaluate_vcp in-place) — 설계 spec

작성일 2026-06-30 · 상태: 설계 승인 대기

## 1. 배경·목적

`vcp-audit`로 미너비니 VCP 정답 예시 6건을 대조한 결과(메모리 [[vcp-book-examples-gaps]]),
현 검출기 `evaluate_vcp`(v1)는 **진짜 VCP를 거의 못 알아본다**(vcp_detected 1/6, 돌파 0/6).
실패가 두 곳으로 좁혀졌다: **베이스·피벗 탐지**와 **돌파 정의**. 이 spec은 그 둘을
고치도록 **기존 `evaluate_vcp`를 직접 수정**하고, 6예시 + 70종목으로 검증한 뒤
머지하는 설계다.

## 2. 확정된 결정 (사용자)
- **패키징**: 별도 v2 안 둠 — **기존 `evaluate_vcp`(vcp.py)를 직접 수정(in-place)**. 변경이
  find-vcp/find-vcp-history/vcp-audit에 즉시 반영되므로 **회귀 가드가 핵심**(머지 전 검증 필수).
- **돌파 거래량**: **터짐 필수**(책 충실). 조용한 돌파(메리츠)는 돌파 신호 미발화 수용.
- **피벗 = 최소저항선** = *마지막(가장 타이트한) 수축의 고점* (lookback 단순 최고가 아님).
- **적응형 수축 임계**: 베이스 변동폭에 비례, **하한 없음**(타이트 수축 최대 포착).
- **성공 기준**: 메리츠 뺀 **5예시(데브시스터즈·한솔케미칼·다올1·2·켐트로스) 인식+돌파** +
  70종목 회귀 가드. 메리츠는 인식되면 좋으나 게이트 아님.

## 3. 현 v1의 실패 지점 (고칠 대상)
| v1 규칙 | 문제 | 예시 |
|---|---|---|
| base_start = lookback 내 최고 *종가* | 횡보 상단이 최근 고점이면 베이스 거의 없음 → base_too_short·피벗 None | 5/6 vcp=False |
| ZigZag 고정 8% | 타이트 횡보(폭<8%)는 수축 0개 | 한솔·켐트로스 |
| 거래량: 후반1/3 평균 < 전반1/3 평균 | 50일선 기준 아님, 수축별로 안 봄 | — |
| 돌파: 종가>피벗 & 거래량≥1.4×베이스평균 | 낡은 피벗·반복발화·음봉허용 | 0/6 |

## 4. v2 설계 — 4가지 수정

### 4.1 베이스·피벗 (횡보 천장 = 최소저항선) — 인식 1/6 표적
- 적응형 ZigZag(§4.2)로 lookback 스윙 추출.
- **수축 수열 = 끝쪽의 *수렴하는* (고→저) 수축 연쇄.** base_start = 그 연쇄 *첫* 수축의 고점 인덱스.
- **피벗(최소저항선) = 횡보 천장 = `max(closes[base_start : 마지막 봉 제외])`** — 베이스(돌파 직전까지)
  의 *종가* 최고치. 주식이 뚫고 올라가는 그 횡보 구간의 저항 천장이다.
  - **안정 레벨**: 평가당 *한 번만* 산출한다(바마다 갱신하는 러닝맥스 아님). 러닝맥스를 쓰면
    closes[-2]≤피벗이 항상 참이 되어 첫돌파 가드(§4.4-1)가 무효화된다(과거 회귀의 원인).
  - **마지막(돌파) 봉 제외**: 천장은 돌파 직전까지의 구간으로 고정 — 돌파 봉 자신이 천장을 끌어올리지 않게.
  - **종가 기준(고가 아님)**: 장중 스파이크 고가(예: 한솔케미칼 ~309,500)를 천장으로 오인하지 않도록.
    종가 천장이 5예시 모두 돌파일 첫돌파를 정확히 발화시킨다(고가 기준은 한솔·다올1차 첫돌파 실패).
  - 마지막 수축 고점(chain pivot)이 아니라 베이스 전체 천장을 쓰므로 '계단식 VCP'(우측 회복이 이미
    마지막 수축 고점을 넘은 경우)에서도 진짜 저항선 최초 돌파를 정확히 잡는다.
- 베이스 = base_start ~ b1(현재).

### 4.2 적응형 수축 — 한솔·켐트로스 0개 표적
- ZigZag 임계 = **베이스 변동폭에 비례하는 적응값**(하한 없음). 후보 형태(plan서 6예시로 보정):
  `thr% = k × (베이스 일간변동성 또는 base_range%)`. 타이트 베이스면 thr이 작아져 미세 수축 포착.
- 수축 성립: **2~6회 + 단계별 비확장(tol)** + **순(net) 수축**(마지막 수축 깊이 < 첫 수축 깊이).
  ← 이 조건들이 적응 임계의 잡음 필터 역할(하한 없는 대신).
- **순수축 정밀도 게이트**: 단계별 tol(1.15)만으로는 깊이가 평탄/확장하는 가짜 수축
  (예: 기가비스 `[11.8, 13.4, 13.2, 14.2]` — 끝이 더 깊음)을 통과시킨다. `depths[-1] < depths[0]`
  (마지막 수축이 첫 수축보다 실제로 얕음)을 추가로 요구해 이런 거짓양성을 탈락(변동성 *수축*의 본질).
  - *거래량 마름 평균(<1.0) 게이트는 채택 안 함*: 활발히 회복 중인 진짜 VCP(한솔 mean 1.49, 켐트로스
    1.27)가 우측 거래량이 높아 같이 탈락하므로 5예시 보존과 양립 불가. dry_max도 0.82 아래로 못 내림
    (켐트로스 dry_min=0.816가 천장). 정밀도 lever는 순수축 게이트가 담당.

### 4.3 거래량 게이트 (50일선 기준) — 책 충실
- 모든 거래량 판정을 **거래량 50일 이동평균** 기준으로(현 후반/전반 폐기).
- **하드 게이트 = 우측 마름**: 베이스 우측(마지막 수축 부근)에 거래량/50선 ≤ dry_max(예 0.7)인
  바 ≥1개. (audit서 6/6 작동 — 견고한 축)
- **수축별 거래량 감소는 *소프트*(점수·근거)로만**, 하드 게이트 아님. (데브시스터즈는 수축별
  거래량이 안 줄지만 진짜 VCP → 하드 게이트로 두면 정답을 탈락시킴.)

### 4.4 돌파 (status) — 0/6 표적, 거래량 터짐 필수
첫돌파일 1회만 발화:
1. **첫돌파**: 전일 종가 ≤ 피벗 AND 당일 종가 > 피벗,
2. **양봉**: 당일 종가 > 시가,
3. **거래량 터짐**: 당일 거래량 ≥ 50일선 × `breakout_vol`(예 1.4),
4. **피벗 근접**: 돌파 시 (종가−피벗)/피벗 ≤ near%(낡은 피벗 반복발화 차단).
- 피벗이 §4.1의 최소저항선이라 첫돌파는 자연히 근접에서 발생.
- *조용한 돌파(메리츠)는 조건3을 못 넘어 미발화 — 의도된 수용.*

### 4.5 반환 스키마
**반환 키는 기존 그대로 유지**(vcp_detected, num_contractions, contractions, base_length_days,
base_depth_pct, pivot_price, pct_to_pivot, volume_dryup_ratio, tightness_pct, status,
swings, reason, entry_ready) — find-vcp/find-vcp-history/vcp-audit가 그대로 읽도록.
내부 로직만 바뀐다. 근거 필드 추가는 가능하나 기존 키 제거·이름변경 금지.

## 5. 검증 하네스
v1을 직접 고치므로, **수정 전 베이스라인을 먼저 캡처**해 두고(이미 보유: vcp-audit의 6예시
결과 = 인식 1/6·돌파 0/6, find-vcp 70종목 산출 = 통과/분포) 수정 후와 비교한다.
- **6예시 회귀**: 수정 후 `vcp-audit`(같은 검출기를 쓰므로 자동 반영) 재실행.
  - 성공 = 5예시(메리츠 제외) **vcp_detected=True** AND **돌파가 정답일 근처(±N일) 발화**.
  - 메리츠: vcp_detected 보고(이상적 True), 돌파 미발화 허용.
- **70종목 회귀 가드**: 수정 후 find-vcp 재실행 → **통과 수가 베이스라인 대비 폭증하지 않음**
  (거짓양성 억제) + actionable/breakout 분포가 비합리적으로 늘지 않음.
- 임계값(k, dry_max, breakout_vol, near, tol)은 **6예시·70종목으로 보정**.

## 6. 구성 요소
- **`scripts/canslim_lib/vcp.py` 직접 수정**: `evaluate_vcp` 내부의 베이스·피벗·수축·거래량·돌파
  로직 교체. 새 순수 부품(`adaptive_zigzag`, `find_contraction_chain`, `detect_pivot`,
  `is_breakout`)을 같은 파일에 추가(기존 `zigzag`는 다른 곳서 쓰면 유지, 안 쓰면 제거 가능).
  `volume_ma`(vcp_audit)·`_mean` 재사용.
- 기존 `tests/test_vcp.py`의 합성 테스트는 새 로직에 맞게 갱신(반환 키는 동일하므로 구조는 유지).
- **검증**: 수정 후 `vcp-audit`(자동 반영) + find-vcp 재실행으로 §5 비교. 필요시
  `public/data/vcp-v2-validation.json` 같은 비교 산출 추가.

## 7. 성공·머지 기준
- **성공(머지 가능)** = 수정 후 5예시 인식+돌파 통과 + 70종목 회귀 가드 통과(임계 보정 포함).
  in-place라 별도 스왑 없음 — 검증 통과가 곧 머지 조건.
- 검증 통과 후 사용자에게 수정 전/후 비교(6예시·70종목)를 보고하고 **머지 승인**을 받는다.
- 동작이 바뀌는 트렌드/find-vcp 스펙·스킬 문구는 [[doc-logic-sync]] 따라 같은 라운드에 갱신.

## 8. 검증 계획
- `vcp.py` 새 부품 단위 테스트: adaptive_zigzag(타이트 시계열서 수축 포착), 수축연쇄/피벗
  선정, is_breakout(첫돌파·양봉·거래량·근접) 합성 시계열로. 기존 test_vcp.py 갱신.
- 6예시 회귀: 5예시 인식+돌파 통과 확인(콘솔/JSON).
- 70종목 비교: 수정 전 베이스라인 대비 통과 수·분포 보고, 폭증 없음 확인.

## 9. 미해결/후속
- 적응형 임계 정확한 식·계수는 plan서 6예시로 보정(경험적).
- 머지 후 find-vcp 재실행·프로덕션 반영은 별도(make-hero/배포).
- 메리츠형 *조용한 돌파*를 별도 신호로 잡을지는 차후(프로젝트 5신호 연구와 접점).
