# 단기-창 거래량 스파이크 진입 신호 — 설계

작성: 2026-07-09

## 배경 & 목표

봇의 거래량 신호는 **누적**(아침 전체 거래량/평소)이라 굼떠서, "이른 돌파 + 늦은 거래량" 승자를 간발로 놓친다(코스맥스엔비티 7/9: 매수존 이탈 ~09:35, 누적이 1.5 넘은 건 09:43 — 신뢰 데이터로 확인). **최근 짧은 창의 거래량 버스트에 즉각 반응하는 신호**를 더해 이런 돌파를 잡는다.

## 신호 정의

`spike_pace = 최근 W분 거래량 / (avg50 × [C(t) − C(t−W)])`
- `C(t)` = 평소 이 시각까지 하루 거래량 누적비율(vol_curve). `C(t)−C(t−W)` = 평소 최근 W분에 나오는 비율.
- 즉 **"지금 최근 W분이, 평소 이 시각 W분보다 몇 배 거래됐나."**
- 시작값: **W = 5분, 문턱 S = 3.0.**

## 봇 결합 (기존을 안 부숨)

```
매수 = 피벗+3% 이내 AND (누적 pace ≥ VOL_PACE_MIN  OR  spike_pace ≥ SPIKE_MIN)
```
- `evaluate_entry`에 `spike_pace`(옵션) 인자 추가. None이면 기존 동작(누적만) — 하위호환.
- 슬롯·추격·미보유 등 나머지 게이트는 그대로.

## 구현 범위 (이번)

1. `strategy_params`: `SPIKE_MIN = 3.0`, `SPIKE_WINDOW_MIN = 5` 추가.
2. `vol_curve.window_vol_frac(hhmmss, window_min)`: `C(t) − C(t−W)` 반환.
3. `signals.evaluate_entry`: `spike_pace=None` 인자 + OR 경로.
4. **일러스트 검증 스크립트**: 신뢰 데이터(당일 TR)로 코스맥스엔비티 7/9 + 오늘 종목에서 "스파이크 게이트가 매수존 안에서 발화하나(누적이 놓친 걸 잡나)" 확인.

## 범위 밖 (다음)

- `replay.py`/`runner.py` 실제 통합(롤링 창 계산) — 일러스트로 유망함이 확인된 뒤.
- 통계적 순효과(승자↑ vs 패자↑) — 신뢰 데이터를 여러 날 쌓은 뒤 백테스트.
- 문턱 S·창 W 최적화.
