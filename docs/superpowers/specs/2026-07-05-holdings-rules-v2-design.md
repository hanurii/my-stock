## 목적

이미 동작 중인 보유 종목 매도규칙 점검(`screen_holdings_feedback.py` + `sell_rules.py` +
`sepa-holdings-feedback.json` + 페이지 섹션)을 두 방향으로 확장한다.

1. **정식 스킬화**: `/sepa` 파이프라인을 돌릴 때마다 자동 실행되도록 `check-holdings`
   스킬을 신설하고 오케스트레이터(`sepa` 스킬)에 편입한다.
2. **규칙 강화**: 미너비니의 돌파 후 위반 개념을 실제 책 예시(오라클)로 보정해
   규칙을 재정의한다.

근거: 마크 미너비니·데이비드 라이언의 "돌파 직후 정상/이상 행동" 정리 + 실데이터
오라클 2건(아래). 사용자 확정 사항은 각 절에 반영.

## 오라클 근거 (Tiingo 실데이터, 2014 상반기)

규칙 임계값을 실제 사례로 못 박았다. 원시 일봉은 Tiingo(상폐주 이력 보존)에서 확보.

### WAGE (WageWorks, 2014-03) — "조용한 돌파 → 시끄러운 붕괴"

- 돌파: 03-18 종가 63.68, 거래량 148,400주 = 50일 평균의 **0.45배** (극저거래량 돌파).
- 붕괴: 03-20부터 **저가 기준 저점경신 6일 연속**. 거래량이 붙은 구간은 03-25·26·27
  (50일 평균의 1.06·1.35·1.26배). 03-24 종가가 20일선 이탈(돌파 4거래일 뒤).
- 책 판정: "적은 거래량 저점경신은 우려 아님, **거래량 붙은 저점경신은 명백한 위반**."

핵심: 저점경신은 **저가 기준**으로 예민하게 보되, **거래량**으로 진짜/가짜를 가른다.
03-20(0.75배)·03-24(0.85배)처럼 평소보다 적은 거래량 저점경신은 봐주고,
03-25~27처럼 평균 이상 거래량이 붙은 3연속 저점경신을 위반으로 잡는다.

### OUTR (Outerwall, 2014-03) — "피벗 근처 즉사(거래량 비대칭)"

- 돌파: 03-04 피벗(≈72.7) 상회, 거래량 1,069,580주 = 평균의 1.45배(강하지 않음).
- 반전: 03-06 종가 71.15(피벗 아래) 거래량 **2.22배**, 03-07 종가 68.55 거래량 **5.00배**.
- 피벗 돌파 후 겨우 +2% 찍고 하루 만에 대량거래로 되밀림.

핵심: "큰 수익을 냈다가 반납"이 아니라 **피벗 근처에서 대량거래로 실패한 돌파**.
돌파(조용/보통) 대비 되밀림 거래량이 훨씬 큰 **거래량 비대칭**이 실패 신호.

## 확정 결정 요약

- **A (거래량 비대칭)**: 되밀림은 **종가**가 피벗 아래인 날로 보고, 그 날 거래량을
  **50일 평균이 아니라 '돌파일 거래량'과 비교**한다(돌파가 워낙 조용하면 평균 대비로는
  안 잡히기 때문). — 규칙⑥에 통합(아래).
- **B (수익 반납/라운드트립)**: **보류**. OUTR은 라운드트립이 아니라 비대칭 사례였고,
  "+N% 벌었다가 반납"은 미너비니의 스쿼트(반전 회복 여유) 개념과 상충 소지가 있어
  이번 범위에서 뺀다.
- **C (저점경신)**: **저가 기준 + 3거래일 연속 + 각 날 거래량 ≥ 50일 평균**일 때만 위반.
  (v1의 종가 기준 폐기). 거래량 낮은 저점경신은 위반이 아니라 detail의 🟡경고로만 표시.
- **⑥+⑦ 통합**: 스쿼트(피벗 이탈)와 거래량 비대칭은 "종가가 피벗 아래" 라는 **같은 축**
  이므로 하나의 규칙으로 합쳐 중복 카운트를 없앤다. 거래량이 스쿼트/실패를 가르는
  스위치다(책도 이 둘을 하나의 이야기로 다룸). 결과적으로 규칙은 **6개**로 유지.

## 최종 규칙 6개 (sell_rules.py)

입력: 일봉 배열(dates/opens/highs/lows/closes/volumes), 돌파일 인덱스 `bi`, 피벗 가격.
반환: `violation` / `pass` / `pending`(날 더 필요) / `na`(판정 불가) + 한 줄 detail.
50일 평균 거래량 = 판정일 직전 50거래일 평균(판정일 제외). 소프트 경고(🟡)는 별도
상태를 만들지 않고 `pass` + detail 텍스트로 표현한다(v1 관례 유지).

| # | id | 상태 | 판정 기준 |
|---|----|------|-----------|
| 1 | low_volume_breakout | 유지 | 돌파일 거래량 < 50일 평균 → 위반. 1.0~1.5배는 통과 + "정상 돌파(1.5배+) 미달" 명시 |
| 2 | heavy_volume_pullback | 유지 | 돌파 후 종가가 전일 종가보다 낮으면서 거래량 ≥ 평균×1.5 인 날이 있으면 위반(가장 심한 날 detail) |
| 3 | **consecutive_lower_lows** | **개편** | 돌파 후, **저가 < 전일 저가**(저점경신)이고 **거래량 ≥ 50일 평균**인 날이 **3거래일 연속**이면 위반. 저점경신 3연속이지만 거래량이 낮은 날이 섞이면 위반 아님 + detail에 "🟡경고: 저점경신 N회(거래량 낮음)" |
| 4 | close_below_ma | 유지(보강) | 돌파 후 종가 < 20일선 → 위반. 종가 < 50일선 + 거래량 ≥ 평균×1.5 → detail "심각(50일선+대량)". detail에 "돌파 N거래일째" 표기 보강 |
| 5 | weak_days_dominant | 유지 | 돌파 후 5거래일 이상 지나야 판정. 하락일 > 상승일 **또는** 나쁜 마감 > 좋은 마감 → 위반. 나쁜 마감 = 종가 < (고+저)/2. 고가=저가·보합일은 세지 않음 |
| 6 | **breakout_failure** | **⑥+⑦ 통합** | 돌파 실패(스쿼트+거래량 비대칭). 아래 판정표 |

### 규칙6 breakout_failure 판정 (스쿼트 통합)

피벗과 돌파를 기준으로 순서대로 판정한다. `SQUAT_GRACE_DAYS = 10`(약 2주).

1. 피벗이 없거나 **돌파가 확인되지 않음**(돌파일이 매수일로 추정) → **na** (돌파 없이는
   돌파 실패도 없음).
2. 돌파 후 종가가 **한 번도 피벗 아래로 안 감** → **pass** ("피벗 위 유지").
3. **종가**가 피벗 아래인 날 중 **거래량 > 돌파일 거래량**인 날이 있으면 → **위반**
   ("거래량 동반 돌파 실패 — {날짜} 거래량 {x}배", x = 그 날 거래량 / 돌파일 거래량).
   **유예 무시**(거래량이 확정 신호). 이후 피벗 위로 회복했더라도 한 번 발생하면 위반.
4. (거래량 조용) **최근 종가가 피벗 아래**이고 돌파 후 경과 ≤ 10거래일 → **pass**
   + detail "🟡 반전 회복 관찰중 (D+n/10)". (위반 아님 — 성급한 손절 방지)
5. (거래량 조용) 최근 종가가 피벗 아래이고 경과 > 10거래일 → **위반** ("유예 초과 —
   피벗 회복 실패").
6. (거래량 조용) 피벗 아래로 갔었지만 **최근 종가가 피벗 위 복귀** → **pass**
   ("스쿼트 후 반전 회복").

"거래량 > 돌파일 거래량"은 결정 A의 잣대다. 돌파가 조용할수록 되밀림이 상대적으로
시끄러워 비대칭이 잘 잡힌다. 오라클 검증: WAGE 03-24(280,200 > 148,400)에서 3번으로
위반, OUTR 03-06(1,689,175 > 1,069,580)에서 3번으로 위반.

## 종합 신호 (v1 그대로)

우선순위: 🔴 손절(현재가 ≤ 매수가×(1+손절%/100), 규칙 무관 최우선) →
🟠 조기 매도(위반 ≥ 1개, 위반 개수 함께 표시) → 🟢 정상 보유(위반 0).
`pending`/`na`/🟡경고(pass)는 위반 개수에 넣지 않는다.

**겹침에 대한 명시**: ②③④⑤는 같은 하락을 서로 다른 렌즈(대량 하락 / 흘러내림 /
선 이탈 / 전반적 약세)로 본다. 대량 하락일이 저점경신·선 이탈과 겹치면 여러 규칙이
같이 켜질 수 있다. 이는 의도된 것으로, 독립적 이유가 여럿일수록 실제로 더 위험하다.
단 위반 개수는 "독립 실패 수"가 아니라 "겹칠 수 있는 방증 수"로 해석한다. ⑥+⑦ 통합으로
가장 큰 중복(피벗 이탈 이중 카운트)은 제거했다.

## 페이지 영향: 최소 (라벨 2줄만)

`SepaHoldingsSection.tsx`는 `rules[]`를 ✓(pass)/✗(violation)/―(pending·na) + detail로
**일반 렌더**하고, 새 상태를 안 만들므로(🟡은 pass+detail) 렌더 로직은 무접촉이다.
다만 페이지에 **규칙 id → 한글 라벨 매핑(`RULE_LABELS`)** 이 있어(계획 작성 중 발견),
id·라벨을 정정해야 한다(그대로 두면 라벨이 깨지거나 의미가 틀림):

- `consecutive_lower_closes: "③ 연속 저저점(종가)"` → `consecutive_lower_lows: "③ 연속 저저점(거래량)"`
- `squat: "⑥ 스쿼트(피벗 복귀)"` → `breakout_failure: "⑥ 돌파 실패(스쿼트)"`

이 2줄 외 페이지 수정은 없다. 규칙 id 소비처는 소스(sell_rules.py)·테스트·이 라벨맵
3곳뿐(grep 확인). 규칙 함수명·id는 의미에 맞게 rename한다
(`rule_consecutive_lower_closes`→`rule_consecutive_lower_lows`, `rule_squat`→
`rule_breakout_failure`).
