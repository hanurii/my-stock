1. API ENDPOINTS AND DATA FORMAT

A. Company Submissions API
Endpoint: https://data.sec.gov/submissions/CIK{CIK}.json
Example: https://data.sec.gov/submissions/CIK0001067983.json (Berkshire Hathaway)

Required Header:
  User-Agent: my-stock-app admin@example.com

Response Structure:
  - JSON format with nested objects
  - Top-level keys: cik, entityType, name, tickers, filings
  - filings.recent contains parallel arrays:
    * form: Array of form types
    * accessionNumber: Array of accession numbers
    * filingDate: Array of filing dates (YYYY-MM-DD)
    * primaryDocument: Array of primary document paths

2. FILING DOCUMENT STRUCTURE

A. Primary Document (primary_doc.xml)

Root Element: <edgarSubmission>

Key Sections:
  headerData:
    - submissionType: "13F-HR"
    - filerInfo: Contains CIK and period of report
  formData:
    - coverPage: Report date, filer name/address, report type
    - signatureBlock: Signer info, title, signature date
    - summaryPage: 
      * otherIncludedManagersCount: Number of other fund managers
      * tableEntryTotal: Total number of holdings (110 in this case)
      * tableValueTotal: Total market value in cents (274160086701)
      * otherManagers2Info: List of co-managers with file numbers

B. Information Table (Holdings Data)

File: {accession_number_without_dashes}.xml (e.g., 50240.xml)

Root Element: <informationTable>

Each holding is an <infoTable> element with:
  - nameOfIssuer: Company name (uppercase, e.g., "ALPHABET INC")
  - titleOfClass: Stock class (e.g., "COM", "CAP STK CL A")
  - cusip: 9-digit CUSIP identifier
  - value: Market value in dollars
  - shrsOrPrnAmt:
    * sshPrnamt: Number of shares (integer)
    * sshPrnamtType: Type code ("SH" for shares)
  - investmentDiscretion: Discretion code (e.g., "DFND")
  - otherManager: Comma-separated manager sequence numbers
  - votingAuthority:
    * Sole: Shares with sole voting authority
    * Shared: Shares with shared voting authority
    * None: Shares with no voting authority

3. DATA FORMAT DETAILS FOR PARSER

Field Types:
  String fields: nameOfIssuer, titleOfClass, cusip, investmentDiscretion
  Numeric fields: value (dollars), sshPrnamt (shares), voting counts
  Array fields: otherManager (comma-separated sequence numbers)

Data Characteristics:
  1. No decimal places: Monetary values are integers in dollars
  2. Multiple entries per issuer: Same CUSIP appears multiple times (different managers)
  3. Hierarchical structure: Holdings split by manager participation
  4. Total aggregation: Sum all <value> elements to verify tableValueTotal
  5. Accession format: NNNNNNNNNN-YY-NNNNNN (10 digits, dash, 2 digits, dash, 6 digits)

4. URL CONSTRUCTION

  Submissions API: https://data.sec.gov/submissions/CIK{CIK}.json
