# 경영진 품질 자동 분류 — 어떻게 점수를 매기는가

> C 페이지 점수 축 ④ (경영진 7점) 에 들어가는 분류를 사람 손 없이 자동으로 매기는 규칙.
> 2026-05-21 초안. 1단계 필터 통과 종목 약 194개를 대상으로 분기마다 자동 갱신.

## 왜 자동 분류가 필요한가

C 페이지의 4축 점수 중 경영진(7점)은 워치리스트의 사람 라벨링에 의존하고 있었습니다. 워치리스트는 *저평가 배당주* 맥락의 81종목만 분류돼 있어서, C 페이지에 노출되는 194종목 중 113개 이상이 "미분류 = 0점" 처리됐습니다. 그 결과 만점에서 -7점 깎인 종목이 절반 이상.

→ **DART 공시를 자동으로 긁어 규칙 기반으로 분류**합니다. 사람 검토 없이 즉시 점수에 반영. 다만 규칙이 틀리면 좌표 일괄 왜곡되므로, 이 문서를 보고 사용자가 피드백 → 규칙 조정의 사이클로 운영.

---

## 한 줄 요약

**최근 5년간 그 회사가 주주를 어떻게 대했는가**를 DART 공시로 본다.
- 자사주를 소각했나 (주주 지분 영구 증가) → 점수 +
- 유상증자·전환사채를 자주 찍었나 (주주 희석) → 점수 −
- CEO 가 자주 바뀌었나 (경영 불안정) → 점수 −
- 감사보고서 의견거절·분식·내부거래 사고가 있었나 → 즉시 최저 등급

이 점수를 종목별로 합산해 3등급(우수 / 전문 / 저조) 으로 떨어뜨립니다.

---

## 6가지 시그널과 가중치

가중치는 *연속 점수* 로 합산하고, 마지막에 임계값으로 분류합니다.

### ① 자사주 소각 (강한 긍정 신호) — 건당 +3점

소각은 *영구* 주식 수 감소 → 기존 주주 지분이 자동으로 늘어남. 가장 강한 주주환원 시그널.

- **DART 공시 키워드**: `자기주식소각결정`, `자기주식소각`
- **수집 범위**: 최근 5년
- **가중치**: 1건당 +3점 (캡 없음 — 자주 할수록 좋음)
- **이유**: 자사주 매입 후 *금고에 묶어두기만* 하면 언제든 다시 풀릴 수 있지만, 소각은 되돌릴 수 없음.

### ② 자사주 매입 (약한 긍정 신호) — 건당 +1점

매입만 하고 소각 안 한 경우. 일시적 수급 효과 + 향후 소각·임직원 보상 활용. 소각 없는 매입은 약한 신호.

- **DART 공시 키워드**: `자기주식취득결정`
- **수집 범위**: 최근 5년
- **가중치**: 1건당 +1점 (캡 3점)
- **주의**: 매입 후 동일 기간 내 소각 공시가 따라오면 ①번에서 이미 +3 잡힘 → 중복 계산 안 함

### ③ 유상증자 (강한 부정 신호) — 건당 −2점

신주 발행으로 기존 주주 지분 희석. 제3자 배정 / 주주배정 / 일반공모 모두 동일 가중치.

- **DART 공시 키워드**: `유상증자결정`
- **수집 범위**: 최근 5년
- **가중치**: 1건당 −2점 (캡 없음)
- **예외**: M&A 자금조달 목적 등 *합리적* 증자도 있지만 자동 판별 어려우니 일단 일괄 감점. 사용자 피드백으로 화이트리스트 운용 가능.

### ④ 전환사채(CB) / 신주인수권부사채(BW) — 건당 −1.5점

잠재 희석. 발행 시점엔 즉시 희석 없지만 전환·행사 시점에 신주 발행 가능. 발행 자체로도 *부채로 자금조달 안 하고 주식으로 떠넘기는* 시그널.

- **DART 공시 키워드**: `전환사채권발행결정`, `신주인수권부사채권발행결정`
- **수집 범위**: 최근 5년
- **가중치**: 1건당 −1.5점

### ⑤ CEO 잦은 교체 — 5년 3회 이상 시 −3점

장기 재임 = 일관된 비전. 잦은 교체 = 경영 불안정·내부 갈등.

- **DART 공시 키워드**: `대표이사변경`, `임원·주요주주 특정증권등 소유상황보고서` 중 대표이사 신규 선임
- **수집 범위**: 최근 5년
- **규칙**: 5년 누적 3회 이상 변경 시 −3점 (한 번에 깎음, 횟수별 누진 아님)
- **이유**: 5년에 2번까지는 정상적 승계로 봄. 3번부터 명백한 불안정.

### ⑥ 감사 사고 / 분식 / 형사처벌 — 적발 시 즉시 −10점 (사실상 poor 확정)

가장 무거운 부정. 회계감리 위반·감사의견 거절·횡령배임·내부거래 형사처벌 등.

- **DART 공시 키워드**: `감사보고서 의견거절`, `감사범위제한`, `상장폐지`, `회계처리기준 위반`, `횡령·배임`
- **수집 범위**: 최근 5년
- **가중치**: 1건이라도 적발되면 즉시 −10점 (다른 긍정 시그널과 무관하게 poor 분류)

### ⑦ 현재 자사주 보유 비중 — 5% 이상 시 −1점

소각 안 한 채 금고에 쌓아둔 자사주는 *발행 안 한 주식* 과 비슷한 효과. 5% 이상이면 거버넌스 부담.

- **데이터 소스**: 종목별 IR 페이지(`fetch_integration` 의 자사주 비중)
- **가중치**: 5% 이상 −1점 (한 번만)

## 합산 → 등급 분류

위 7가지 시그널의 가중치를 모두 더한 *총점* 으로 분류합니다.

| 총점 | 분류 | 점수(축 ④) |
|---|---|---|
| **+3 이상** | 우수 (excellent) | 7점 |
| **−2 초과 ~ +3 미만** | 전문 (professional) | 4점 |
| **−2 이하** | 저조 (poor) | 0점 |
| **감사 사고 1건+** | 저조 (poor) | 0점 (자동 강등) |

### 예시 시나리오

**케이스 A — 우수 (7점)**
- 자사주 소각 2회 (+6)
- 자사주 매입 1회 (+1)
- 유상증자·CB 없음
- CEO 5년 1회 교체
- 자사주 비중 1%
- **총점 +7 → 우수**

**케이스 B — 전문 (4점)**
- 자사주 소각 0회
- 자사주 매입 1회 (+1)
- 유상증자 1회 (−2)
- CB 없음
- CEO 5년 2회 교체
- 자사주 비중 3%
- **총점 −1 → 전문**

**케이스 C — 저조 (0점)**
- 자사주 소각·매입 없음
- 유상증자 2회 (−4)
- CB 1회 (−1.5)
- CEO 5년 3회 교체 (−3)
- **총점 −8.5 → 저조**

**케이스 D — 즉시 저조 (감사 사고)**
- 다른 시그널 어떻든 무관
- 감사보고서 의견거절 1회
- **자동 저조 (poor)**

## 데이터 소스 — 어디서 무엇을 가져오나

| 시그널 | 소스 | API |
|---|---|---|
| 자사주 소각·매입 | DART | `opendart.fss.or.kr/api/list.json` 의 `report_nm` 키워드 매칭 |
| 유상증자 | DART | 동일 |
| CB / BW | DART | 동일 |
| CEO 교체 | DART | 동일 (`대표이사 변경` 공시) |
| 감사 사고 | DART | 감사보고서 / 공시 변경 / 정정 보고 검색 |
| 자사주 보유 비중 | 네이버 IR | 종목별 integration 페이지의 `자사주 비중` |

DART 검색 범위: **최근 5년** (오늘 기준 −5년 ~ 오늘).

## 한계점 — 어디서 틀릴 수 있나

1. **합리적 증자도 일괄 감점** — M&A 자금조달, 운영자금 조달 등 *건강한* 증자도 동일 가중치로 깎임. 향후 화이트리스트 도입 가능.
2. **창업자 직접 경영 같은 *질적* 시그널 미반영** — 워치리스트의 "창업자·오너 직접 경영" 같은 정성 평가는 자동화 불가. 사용자가 강하게 원하는 종목이 자동 분류로 *전문* 에 머무를 수 있음.
3. **DART 키워드 매칭의 노이즈** — 공시 제목에 키워드가 들어가도 실제 본문이 다를 수 있음. 예: "자기주식취득결정" 의 *철회* 공시도 동일 키워드 포함 → 별도 정정 공시 추적 필요.
4. **CEO 교체 = 부정?** — 평범한 세대교체도 −3점 깎임. 5년 3회 임계가 너무 엄격할 수도. 사용자 피드백으로 조정 가능.
5. **신규 상장 종목 (5년 미만)** — 데이터 부족으로 보수적 분류. 신규 종목은 *전문* 으로 기본 분류.

## 갱신 주기

- **분기마다 1회 자동 실행** (`scripts/build_management_quality.py`)
- 1단계 필터 통과 종목 목록 변동 시 → 새 종목만 추가 수집
- 결과는 `public/data/management-quality.json` 에 저장
- C 점수 계산 시 이 파일을 우선 로드, 미등록 종목만 워치리스트로 보조

## 사용자 피드백 → 규칙 조정 흐름

이 문서가 *명세* 역할을 합니다. 사용자가 자동 분류 결과를 보고 의문이 들면:

1. 의문 종목의 *시그널 분해* 를 확인 (분류 JSON에 시그널 detail 포함됨)
2. 어떤 시그널이 잘못 잡혔는지 식별
3. 이 문서의 해당 시그널 가중치·임계값을 조정
4. 다음 자동 실행에서 반영

가중치는 *건당 +3, 건당 −2* 같은 단순 정수로 잡혔습니다. 미세 조정 시 0.5 단위로 변경 가능.

### 자주 나올 피드백 예시

- *"이 종목 우수인데 전문으로 떨어졌네"* → 자사주 소각 가중치 +3 → +4 로 올리거나, 임계값 +3 → +2 로 낮추기
- *"이 종목 명백히 저조인데 전문이네"* → 유상증자 가중치 −2 → −3 으로 강화, 또는 CB 가중치 강화
- *"감사 사고 5년 지났는데 아직도 저조?"* → 감사 사고 검색 범위 5년 → 3년으로 줄이기

## 출력 데이터 구조

`public/data/management-quality.json`:

```json
{
  "generated_at": "2026-05-21",
  "rule_version": "v1",
  "stocks": {
    "005930": {
      "name": "삼성전자",
      "quality": "excellent",
      "total_score": 7.5,
      "signals": {
        "buyback_cancel_count": 2,
        "buyback_acquire_count": 1,
        "rights_issue_count": 0,
        "cb_bw_count": 0,
        "ceo_change_count": 1,
        "audit_issue_count": 0,
        "treasury_ratio": 1.2
      },
      "evidence": [
        { "date": "20231115", "title": "자기주식소각결정", "score_impact": "+3" },
        { "date": "20240228", "title": "자기주식소각결정", "score_impact": "+3" }
      ]
    }
  }
}
```

`evidence` 필드에 각 시그널의 실제 공시 제목·날짜·점수 영향을 박아둬서, 의문이 들 때 즉시 추적할 수 있게 합니다.
