# /stocks/canslim/i 페이지 — 어떤 종목이 보이는가

> "이 페이지에 뜬 종목, 도대체 무슨 기준으로 골라진 거지?" 한 문서로 답하기 위해.
> 2026-05-21 초안. C/A/L/N/S 페이지 문서와 짝.

---

## 1. 한 줄 요약

> **"진짜 큰 손이 이 종목을 사고 있는가"** — 5%룰 대량보유 보고와 일별 기관 순매매로 *기관의 양·추세·신규 편입* 을 검증한 종목.

오닐의 CAN SLIM 에서 **I = Institutional Sponsorship (기관 투자가의 뒷받침)** 입니다.

- **C/A/N/S** — 종목 *자체* 의 실적·신고가·체질.
- **L** — 시장 *상대* 강도 (RS 80+).
- **I 페이지 (여기)** — *누가* 그 종목을 사고 있는가. 기관이 사면 매수 자격, 빠져나가면 매수 금지.

> 오닐: *"적어도 여러 기관이 보유해야 한다."* + *"가장 최근 분기에 비중 있게 신규 편입된 종목이 훨씬 중요하다."* + *"기관이 빠져나가는 종목은 매수하지 말라."*

---

## 2. 입력 모집단 — L 통과 종목

```
[L 통과 종목]  ─→  보고자 분류 + 매매 누적  ─→  4 게이트 + 4 경고  ─→  통과/회색
```

- L 페이지에서 *통과* 한 종목만 I 입력 (`can-slim-l-candidates.json`).
- L 미통과(RS < 80) 종목은 I 평가도 안 됨 — 책 원칙: 상대 강도 없는 종목은 기관 뒷받침 의미 없음.
- I 자체에서 DART majorstock + 네이버 일별 기관 매매를 새로 수집.

> **코드 격리** — I 평가는 C/A/N/S/L 결과를 *사용하지 않음*. 이전 단계가 비어 있으면 I 도 비어 있음.

## 3. 데이터 소스 — 한국식 한계와 보정

### 왜 미국식 그대로 못 쓰나

오닐 책은 *13F 분기 공시* 를 전제로 *"기관 보유 종목 수 + 신규 편입 비중 + 매니저 등급"* 을 직접 본다고 가정합니다. 한국엔 13F 같은 펀드별 전수 공시가 없습니다.

| 미국 (책 전제) | 한국 (우리 환경) |
|---|---|
| 13F 분기 공시 | DART 5%룰 — *5% 이상 + 1%p 변동* 시에만 보고 |
| 펀드별 전 종목 비중 | 5% 미만 보유는 *불가시* |
| 매니저 등급 (S&P, Lipper 등) | fundguide.net 분기/연간 Top 10 — *수동 입력 + 상대평가* |

### 우리가 쓰는 3개 소스

| 소스 | 무엇을 보는가 | 한계 |
|---|---|---|
| **DART majorstock** | 5% 이상 보유자의 보고 시계열 (운용사·연기금·계열사·임원) | 5% 미만 보유 잡히지 않음 → 시총 100조+ 메가캡은 시그널 약함 |
| **네이버 일별 기관 매매** (`/item/frgn.nhn`) | 일별 *기관 합계* 순매매. 60일 누적으로 추세 판단 | 기관 *내부* 구분 없음 (운용사·연기금·은행 합산). 백필 페이지 단위로 약 20일/페이지 |
| **fundguide.net Top 10** | 운용사 1년/3년 수익률 순위 → a+/a/a- 상대평가 | Top 10 한정이라 b/c 등급 *불가능*. 분기마다 수동 입력 필요 |

## 4. 평가 구조 — 4 게이트 + 4 경고

### 핵심 원칙

- **이분법** — 게이트 4개 중 *하나라도* 해당하면 회색 처리 (메인 후보에서 배제, 단 페이지에선 *사유와 함께* 노출).
- **점수 없음** — 다른 페이지(C·A·L)와 달리 *통과/미통과* 만. 책 자체가 점수가 아닌 *자격 조건* 으로 기술.
- **경고는 통과에 영향 없음** — 4개 경고는 통과해도 표시. *질적 보조 정보*.

### 노출 게이트 (G1~G4)

| 게이트 | 조건 | 책 근거 |
|---|---|---|
| **G1 기관 매매 두 분기 연속 이탈** | 직전 60일 기관 순매매 < 0 AND 그 전 60일 (60~120일) 도 < 0 | *"꾸준히 기관 매도 → 매수 금지"* |
| **G2 5%룰 지분 1년 감소 + 신규 시그널 0** | 1년간 누적 지분율 변동 < -2.0%p AND 신규/추가매수/재등장 0건 | 양·추세·신규 *세 가지 모두* 죽었다는 신호 |
| **G3 5%룰 이탈 다수** | 1년 내 5%→미만 이탈 보고자 ≥ 2건 | 기관 *연쇄 매도* — 단일 이탈은 우연일 수 있지만 2건+ 는 의미 있음 |
| **G4 기관 뒷받침 완전 부재** | 5%+ 운용사·연기금 0개 AND 60일 매매 ≤ 0 AND 신규 시그널 0건 | "기관이 *전혀* 안 보는 종목" — 오닐: *"적어도 여러 기관이 보유해야"* 미충족 |

### 경고 시그널 (C1~C4)

| 경고 | 조건 | 의미 |
|---|---|---|
| **C1 신규 시그널 0건** | 1년간 strict 신규 + 추가매수 + 재등장 모두 0 | *책 최강조 항목* — "신규 편입이 가장 중요" |
| **C2 직전 분기 큰 하락** | 직전 60일 음수 AND (그 전 60일 양수 OR -50% 이상 가속) | 양수→음수 전환 또는 음수가 더 깊어진 케이스 |
| **C3 분기 추세 악화** | 직전 60일 누적 < 그 전 60일 누적 (10%+ 차이) | 매수세 *둔화* — 아직 양수일 수 있지만 동력 감소 |
| **C4 5%룰 부재 + 60일 이탈** | 5%+ 기관 0개 AND 60일 누적 < 0 | 큰 손도 작은 손도 다 나가는 중 |

## 5. 신규 시그널 3 카테고리

> 책 #4: *"가장 최근 분기에 비중 있게 신규 편입된 종목은 기존 보유보다 훨씬 눈여겨봐야 한다."*

DART 5%룰 시계열에서 *신규 편입* 을 어떻게 잡을지가 핵심입니다. 단일 정의로는 한계가 있어 **3개 카테고리** 로 분리해 *어느 한쪽이라도 활성화* 되면 신규 시그널로 카운트합니다.

| 카테고리 | 정의 | 의미 |
|---|---|---|
| **🆕 strict 신규** (`is_strict_new_1y`) | DART 시계열에서 *첫 등장* 이 1년 이내 + 현재 5%+ | "이 종목을 처음 사기 시작" — 가장 강한 시그널 |
| **➕ 추가매수** (`is_recent_buyer_90d`) | 최근 90일 내 `stkrt_irds > 0` 보고 1건+ + 현재 5%+ | 기존 보유자가 *비중 확대* — 확신 강해진 신호 |
| **🔄 재등장** (`is_returning_after_gap`) | 보고 간격 ≥ 180일 후 재진입 + 마지막 보고 1년 내 + 현재 5%+ | 한 번 정리한 운용사가 다시 들어옴 — 컨빅션 회복 |

### 우선순위 (한 보고자 = 하나의 라벨)

`strict_new > returning > recent_buyer` 순으로 적용. 예: strict 신규이면서 90일 내 추가 매수도 있는 경우엔 *strict 신규* 라벨만 부여.

### 카테고리는 운용사·연기금만

`other` (계열사·임원·기타) 보고자는 *분류 가능 카테고리* 가 아니라서 신규 시그널에 포함되지 않습니다. 책 원칙 — "기관 투자가" 의 신규 편입만 의미.

## 6. 보고자 분류 — 4 카테고리

DART majorstock 의 `repror` (보고자명) 을 키워드로 분류합니다.

| 카테고리 | 키워드 | 예시 |
|---|---|---|
| **🇰🇷 한국 운용** (`korean_am`) | "자산운용", "투자신탁운용", "투자자문" | 미래에셋자산운용, KB자산운용 |
| **🌐 글로벌 운용** (`global_am`) | BlackRock, Capital, Vanguard, Fidelity, StateStreet, Norges, Macquarie, Nomura, Morgan, Goldman, JPMorgan, Schroder, "Asset Management", "Fund", "Advisors", "LP" | BlackRock, Inc., Capital Research and Management |
| **💼 연기금** (`pension`) | 국민연금공단, 사학연금, 공무원연금, 우정사업본부 | 국민연금공단 |
| **기타** (`other`) | 위 3개에 매칭 안 됨 | 계열사·임원 · 개인 대주주 등 |

> **양 점검 (책 #2-a)**: 5%+ 컬럼의 괄호 안 *(한국/글로벌/연기금)* 분해 수치로 한눈에 확인.

### 글로벌 운용 = unrated 라벨

fundguide 는 한국 운용사 한정이라 글로벌 운용은 *unrated 글로벌* 로만 표시됩니다. 단 *5% 이상 보유 자체가 강한 시그널* 이라 점수 페널티 없음. (운용사 관점 페이지 참고)

## 7. 기관 매매 누적 (네이버) — 60일·QoQ 정의

### 왜 60일인가

네이버 일별 기관 매매를 백필 — 1페이지 ≈ 20영업일.

- `pages=3` (기본) → 약 60영업일 ≈ 직전 *1분기*. 오닐: *"분기 보고서 사이 시장 추세 판단"* 의 한국식 근사.
- `pages=12` (옵션) → 약 240영업일 ≈ 1년.

기본은 *직전 60일 + 그 전 60일* 2분기 비교라 `pages=6` 이 권장.

### 추세 지표 3개

| 지표 | 정의 |
|---|---|
| `cum_60d` | 직전 60영업일 기관 순매매 누적 (주) |
| `cum_prev_60d` | 그 전 60영업일 (60~120일) 누적 |
| `trend_qoq` | `cum_60d` vs `cum_prev_60d` 차이가 \|cum_prev\|×10% 미만 → flat, 이상 → improving/deteriorating |

### 게이트에 쓰이는 합성 플래그

| 플래그 | 정의 | 게이트 |
|---|---|---|
| `is_outflow` | `cum_60d` < 0 | C4 |
| `is_consistently_declining` | `cum_60d` < 0 AND `cum_prev_60d` < 0 | G1 |
| `is_sharp_drop_qoq` | 직전 분기 양수→음수 전환 OR 직전 분기 대비 50%+ 더 하락 | C2 |

> **데이터 한계** — 네이버 일별값은 *기관 합산*. 운용사·연기금·은행 구분 없음. 5%룰 시계열 (운용사 식별 가능) 과 *교차* 해서 보강.

## 8. 5%룰 1년 누적 지분 변동

DART 보고서의 `stkrt_irds` (지분율 증감) 을 *1년 내 모든 보고 합산*. 단위 %p.

| 값 | 의미 |
|---|---|
| `+5.0%p` | 1년간 운용사·연기금이 합쳐서 5%포인트 *순매수* |
| `0%p` | 변동 없음 |
| `-2.0%p` 이하 | G2 게이트 — 신규 시그널 0건과 결합되면 회색 |
| `-5.0%p` | 1년간 5%포인트 *순매도* |

> **G2 컷오프가 -2.0%p 인 이유** — 더 엄격하면(예: -5%p) 약한 이탈을 놓치고, 더 느슨하면(예: -1%p) 변동성 잡음에 잡힘. 신규 시그널 0건과 *AND* 결합으로 *추세 + 신규 둘 다 죽음* 만 잡도록 설계.

## 9. 회색 처리 vs 미통과

L 페이지와 달리 I 는 *회색 음영* 으로 처리 — 제외되더라도 *사유와 함께* 메인 테이블에 노출됩니다.

| 상태 | UI | 의도 |
|---|---|---|
| **통과** (`passes_i: true`) | 정상 표시 | 매수 자격 |
| **회색** (`passes_i: false`) | `opacity-55` + 사유 칩 | "왜 떨어졌는지" 즉시 확인 가능 |

> L 처럼 *별도 섹션* 으로 빼지 않은 이유 — I 는 *데이터 누락* (DART 미보고·네이버 백필 실패) 가능성이 더 높아, 사유 노출이 매수 판단 보조에 더 중요.

## 10. 페이지가 보여주는 것

### 10-1. 종목 관점 메인 테이블 (`/stocks/canslim/i`)

| 컬럼 | 의미 |
|---|---|
| **종목명** + 코드 | 회색이면 회색 음영 |
| **5%+ 기관** | 총합 (한국/글로벌/연기금) 분해 |
| **신규 시그널** | strict + 추가매수 + 재등장 합계 |
| **1년 지분 변동** | `total_stkrt_change_1y_pct`. 양수 녹색·음수 적색 |
| **60일 기관 매매** | `cum_60d` (주). 양수 녹색·음수 적색 |
| **분기 추세** | improving/flat/deteriorating |
| **상태** | 통과 / 회색 사유 / 경고 칩 |

행 클릭 시 펼쳐짐:
- 보고자별 시계열 (현재 지분율 · 첫 보고일 · 신규 라벨)
- 1년 내 5% 이탈 보고자 목록
- 60일/직전 60일 누적 + 수집 기간

### 10-2. 운용사 관점 페이지 (`/stocks/canslim/i/managers`)

워치리스트 + L 통과 종목을 *모집단* 으로 한 *운용사 역인덱스*. "이 운용사가 어떤 종목들에 5%+ 들고 있는가" 를 등급별로 모아 보여줍니다.

- **fundguide Top 10** — 1년/3년 수익률 상위 10개 운용사 + a+/a/a- 등급
- **분기별 종합 등급** — 4분기 데이터 합산 가중 점수 (`consolidate_quarterly_to_annual`)
- **운용사 카드** — 등급별로 그 운용사가 보유 중인 모집단 종목 리스트

### 10-3. 평가 기준 안내 박스

페이지 상단에 *4 게이트 + 4 경고* 를 그리드로 항상 표시 — *왜 회색인지* 가 한눈에 들어오도록.

## 11. 자주 묻는 의문

### Q1. L 통과 종목이 왜 I 페이지에 안 떠요?

DART majorstock fetch 실패 또는 네이버 백필 실패 가능성이 가장 큽니다. JSON 의 `fetch_error` 필드에 사유 기록. 데이터 부재는 *제외* 가 아닌 *불가시* 입니다 — 다음 갱신에서 다시 시도됩니다.

### Q2. 5%룰 보고자 0개인데 통과되는 경우가 있나요?

**예.** 시총 100조+ 메가캡(삼성전자·SK하이닉스 등)은 *기관이 5% 미만씩 분산 보유* 하는 게 정상이라 DART 보고가 적습니다. 이 경우:

- 60일 기관 매매가 *양수* 면 통과 가능 (G4 미해당).
- C4 경고 ("5%룰 기관 부재 + 60일 이탈") 가 표시되긴 함.

> 메가캡 한계는 책 자체가 미국 13F 전제이기에 발생하는 *구조적* 문제. 5%룰만으로 모든 종목을 일관 평가할 순 없음을 인지하고, 메가캡은 *네이버 일별 매매* 에 비중을 더 두세요.

### Q3. 신규 시그널이 *strict 신규* 와 *추가매수* 가 같은 1점인가요? 강도 차이 없나요?

**카운트 자체는 같습니다.** 다만 펼침 뷰에서 보고자별 라벨이 다르게 표시 (🆕/➕/🔄). 또한 우선순위는 `strict_new > returning > recent_buyer` 라 strict 가 있으면 그것만 부각됩니다.

책 본문은 *"신규 편입이 가장 중요"* 만 강조하고 추가매수와의 *우열* 까지 정의하진 않습니다. 우리는 *셋 다 신호로 인정* 하고 라벨로 결을 보여주는 방식을 택했습니다.

### Q4. 기관이 *매도하면* 무조건 매수 금지인가요? 한 분기만 매도면 어떡하나요?

**한 분기만 매도면 통과 가능합니다.** G1 은 *두 분기 연속* 음수일 때만 발동. 단:

- 직전 분기만 음수면 C3 (추세 악화) 또는 C2 (큰 하락) 경고 표시.
- 책 *"한 분기 매도여도 경계"* 정신은 *경고 시그널* 로 살림 (게이트는 아님).

> 두 분기 연속을 게이트 조건으로 둔 이유 — 한국 시장은 *분기 결산 일시적 리밸런싱* 매도가 잦아 단일 분기 음수만으로는 *진짜 이탈* 판정이 어려움.

### Q5. 운용사 등급이 *unrated* 인데 그게 매수 결격인가요?

**아닙니다.** unrated 는 *데이터 미수록* 일 뿐 *나쁜 운용사* 가 아닙니다.

- 글로벌 운용사 (BlackRock·Norges 등) 는 fundguide 미수록 → 무조건 unrated 글로벌.
- 한국 운용사라도 fundguide Top 10 밖이면 unrated.

5%+ 보유 자체가 강한 시그널이라 unrated 도 *통과* 자격엔 영향 없음. 단 책 #2-c *"매니저 등급"* 의 정신은 a+/a/a- 운용사의 *집중 보유* 종목을 *우선 검토* 하는 방식으로 살립니다 (운용사 관점 페이지).

### Q6. C 페이지 점수가 높은데 I 페이지에서 회색 처리됐어요. 어떻게 해석하나요?

**전형적인 "실적은 좋지만 기관이 빠지는" 함정입니다.**

| C 점수 | I 상태 | 해석 |
|---|---|---|
| 높음 | 통과 | 실적 + 기관 매수 동시 — 진정한 후보 |
| 높음 | 회색 (G1/G3) | 실적은 좋지만 기관이 빠짐 — *책 기준 매수 금지*. 기관이 보는 *무언가* 가 있다는 의미 |
| 높음 | 회색 (G4) | 메가캡일 가능성. C 점수만으로 판단 |
| 낮음 | 통과 | 실적은 약하지만 기관 매집 중 — *선행 매집* 가능성. 다만 책은 C 가 약하면 매수 금지 |

오닐 명제: *"기관이 매도하는 종목은 어떤 다른 조건도 무의미하다."* — I 회색이면 다른 모든 점수를 *override* 한다고 보세요.

### Q7. 점수가 없으니 종목 간 *우선순위* 는 어떻게 정하나요?

I 자체로는 *통과/회색* 만 가르고, 정렬은 두 가지 기준을 권장:

1. **신규 시그널 합계** 내림차순 — 책 #4 강조: 신규 편입이 가장 중요.
2. **5%+ 기관 수** 내림차순 — 책 #2-a 양 점검.

테이블 정렬 컬럼은 사용자가 조정 가능. 기본 정렬은 L 의 RS 순서를 유지 (앞 단계 우선순위 보존).

### Q8. fundguide 데이터는 어디에서 입력하나요? 자동 수집인가요?

**수동 입력입니다.** fundguide.net 의 분기/연간 Top 10 페이지를 사용자가 분기마다 `public/data/fund-rankings.json` 에 직접 갱신합니다.

- 자동화 안 한 이유: fundguide 페이지 구조 변경이 잦고 *Top 10 한정* 데이터라 robust 한 스크레이퍼 개발 비용 > 수동 입력 비용.
- 미갱신 시 — 등급 lookup 모두 unrated. 페이지 동작엔 영향 없고, 운용사 카드 정렬만 *5%+ 보유 종목 수* 기준으로 자연 fallback.

### Q9. I 하나만 보면 충분한가요?

**부족합니다.** I 는 *수급* 만 검증합니다.

실제 매수 결정은 다음 다섯을 *함께* 봐야 합니다:

- **C** — 현재 분기 실적
- **A** — 연간 실적의 진정성
- **L** — 상대 강도
- **M** — 시장 추세
- **I** — 외인·기관 수급 ← *이 페이지*

특히 *M (시장 추세)* 가 약세장이면 I 통과 종목도 매수 보류 (sell 페이지의 약세장 배너 참고).

## 12. 데이터 갱신·운영 정보

- **선행 조건** — L 결과 (`can-slim-l-candidates.json`) 가 먼저 갱신되어 있어야 함.
- **실행**:
  - `python scripts/screen_canslim_i.py` → `can-slim-i-candidates.json` 생성
  - `--limit N` — 디버그용 N개만 처리
  - `--org-pages 6` — 네이버 백필 페이지 수 (기본 3 = 60일, 권장 6 = 120일)
  - `--merge` — 기존 JSON 머지 (실패 종목만 재시도)
- **소요 시간** — 한 종목당 DART majorstock 1회 + 네이버 백필 3~12회 + 0.3s sleep → L 통과 종목 50개 기준 **약 5~15분** (네이버 페이지 수에 비례).
- **운용사 관점 데이터** — `python scripts/build_manager_portfolios.py` 별도 실행 → `manager-portfolios.json` 생성.
- **fundguide 갱신** — `public/data/fund-rankings.json` 수동 편집. 분기별/연간 raw 입력 시 `consolidate_quarterly_to_annual` 가 자동 가중 종합.
- **코드 격리** — I 의 모든 로직은 `canslim_lib/criteria_i.py` 에 격리. L 코드는 수정하지 않음.
- **핵심 함수** — `classify_reporter`, `analyze_majorstock`, `analyze_org_flow`, `evaluate_i`, `consolidate_quarterly_to_annual`, `lookup_manager_grade`.
- **페이지 표시** — 상단 *생성일* 이 마지막 업데이트 시점. 데이터 미생성 시 안내 메시지 노출.
