# /stocks/canslim/l 페이지 — 어떤 종목이 보이는가

> "이 페이지에 뜬 종목, 도대체 무슨 기준으로 골라진 거지?" 한 문서로 답하기 위해.
> 2026-05-21 v3. A 페이지(`IMPL_canslim_a_page.md`) 와 짝.

---

## 1. 한 줄 요약

> **시장 대비 상대 수익률 = 점수.** 지난 52주 수익률을 KOSPI 시총 상위 300종목과 비교한 *백분위(1~99)* 가 곧 L 점수. 별도 컷오프 없이 *점수 순서대로* 정렬됩니다.

오닐의 CAN SLIM 에서 **L = Leader or Laggard (주도주 또는 소외주)** 입니다.

- **C / A 페이지** — *종목 자체* 의 분기·연간 실적이 좋은가.
- **N / S / I 페이지** — *카탈리스트·수급 체질·기관 매수* 가 받쳐주는가.
- **L 페이지 (여기)** — 시장이 *실제로* 매수하고 있는가. 가격이 그것을 증명하는가.

| C·A 점수 | L 점수 | 해석 |
|---|---|---|
| 높음 | 높음 | 실적도 좋고 시장이 이미 인정 — 가장 강한 후보 |
| 높음 | 낮음 | 실적은 좋지만 *시장이 아직 매수 안 함* — 함정일 가능성 |
| 낮음 | 높음 | 실적 약한데 가격만 강함 — 모멘텀 추격 위험 |

> 오닐: *"정말 애가 탈 정도로 싸게 보이는 주식이라 해도 소외주는 투자 수익을 가져다 주는 경우가 거의 없다."* 실적이 좋아도 *상대적 주가 강도(RS)* 가 낮으면 매수 보류.

## 2. 우리가 보는 한 가지 점수 지표 + 한 가지 라벨 지표

| 지표 | 무엇을 보는가 | 활용 |
|---|---|---|
| **RS = L 점수** (1~99) | 모집단(KOSPI 시총 상위 300) 대비 52주 수익률 *백분위* | 점수 (100점 만점 척도) |
| **52주 수익률** (raw %) | 1년 전 종가 대비 현재가 변동률 | RS 의 원본 수치. 비교 시 참고. |
| **모집단 위치** (`in_universe`) | KOSPI 시총 상위 300 안에 들었는가 | **라벨만** (점수 영향 없음, RS 정확도 보조) |

### 왜 절대 수익률이 아니라 *백분위(RS)* 인가

같은 +30% 수익률이라도 *시장 평균이 −10% 인 해* 와 *+50% 인 해* 에서 의미가 정반대입니다.

- 절대 수익률 → 시장 환경의 노이즈가 끼어 *어떤 시기엔 다 통과, 어떤 시기엔 다 탈락* 이 됨.
- RS 백분위 → *언제 측정해도 상위 20% 는 항상 상위 20%*. 시장 환경에서 분리된 *상대적* 강도만 잡힘.

오닐 책의 핵심 주장 *"주도주를 사고 소외주를 피하라"* 는 본질적으로 *상대 위치* 의 문제이지 절대 수익률의 문제가 아닙니다.

### 모집단 위치 라벨이 왜 필요한가 (점수에는 영향 없음)

코스닥 종목은 모집단(KOSPI 300) 밖에 있어 *직접 백분위 계산* 이 불가능합니다. 우리는:

- KOSPI 300 종목 → 모집단 내. 정렬 인덱스로 RS 1~99 직접 산출.
- 코스닥·KOSPI 301위 이하 종목 → 모집단 외. *해당 수익률보다 낮은 모집단 종목 수* 로 백분위 *추정*.

추정치는 KOSPI 300 분포에 맞춰 환산한 *간접 측정* 이므로, 정확한 비교를 위해서는 `in_universe = false` 라는 라벨이 표에 함께 노출돼야 합니다. **라벨은 점수에 가감 없이 정보 표시 용도** 입니다.

### 데이터 출처

- **모집단 종목 리스트** — KRX KOSPI 시총 정렬 상위 300 (`fetch_stock_list("KOSPI")[:300]`).
- **일봉 차트** — Yahoo Finance 1년 일봉 (`range=1y, interval=1d`). 종가 200개 미만이면 데이터 부족으로 제외.
- **52주 수익률** — `(현재가 − 1년 전 종가) / 1년 전 종가 × 100`. 분기 종가 보정·배당 reinvest 없음 (단순 수익률).

## 3. 점수 체계 개요

### 핵심 원칙 (A 페이지와 동일)

- **탈락 없음** — 어떤 종목도 점수 미달로 페이지에서 사라지지 않습니다. 모두 *점수 순서* 로 줄을 섭니다.
- **컷오프 없음** — "RS 80 이상만 통과" 같은 이분법 없이, 1점부터 99점까지 *연속 척도*.
- **점수 순 정렬** — 한 개의 정렬된 리스트. 등급 라벨은 *분류 정보* 일 뿐.

### 만점 — 99점 (실질 100점 만점 척도)

RS 는 백분위라 *수학적으로 1~99 범위* 입니다 (가장 낮은 종목 1점, 가장 높은 종목 99점). 실질 100점 만점 척도로 다른 페이지 점수와 동일선상 비교 가능.

| 페이지 | 만점 | 의미 |
|---|---|---|
| A | 50점 | 실적의 진정성 |
| **L (여기)** | **99점** | **시장 대비 상대 강도** |

> A 와 L 은 만점 단위가 다르지만 *각자의 척도에서 점수가 곧 가치* 라는 원칙은 동일.

### 데이터 부족 처리

Yahoo 1년 일봉 200개 미만이면 *점수 산출 불가*. 이런 종목은:
- `rs_score = null` 로 표시.
- 페이지에서는 *최하단* 에 "데이터 없음" 섹션으로 노출.
- 신규 상장 종목·거래정지 종목에서 발생.

**"데이터 없음 ≠ 0점."** 평가 불가일 뿐이므로 0점으로 환산하지 않습니다.

## 4. RS 점수 산출 방식

### 4-1. 모집단 선정 — KOSPI 시총 상위 300

| 선택 기준 | 이유 |
|---|---|
| **KOSPI 만** | 코스닥과 변동성·자금 흐름이 달라 한 모집단에 섞으면 RS 가 왜곡 |
| **시총 상위** | 백분위 비교 시 *대표성* — 거래량 미달 소형주가 모집단에 끼면 노이즈 |
| **300종목** | 백분위 1~99 환산 시 *통계적 충분성* 확보 (분포 해상도 약 1%p) |

### 왜 한국형 모집단이 따로 있나

오닐 원전은 *미국 시장 전체* 를 모집단으로 두지만 그대로 적용하면:
- KOSPI + 코스닥 ≈ 2,400 종목. 거래량 미달 소형주가 절반 이상이라 *백분위가 거의 무의미*.
- 코스닥 급등주(테마주 등) 가 RS 99 를 차지하고 *진정한 KOSPI 주도주가 묻힘*.

→ **KOSPI 시총 상위 300** 으로 모집단을 *제한* 하고, 모집단 외 종목은 *추정 RS + 라벨* 로 별도 처리합니다.

### 4-2. 계산 단계

1. **52주 수익률 계산**
   - 모집단 + S 통과 종목 (모집단 외 포함) 모두에 대해 Yahoo 1년 일봉 수집.
   - 종가 200개 미만이면 데이터 부족 → 제외 (모집단에서도, 평가에서도).
   - `return_1y_pct = (last_close − first_close) / first_close × 100`

2. **모집단 백분위 환산 → RS 점수**
   - 모집단을 수익률 오름차순 정렬.
   - 공식: `rs = round(1 + 98 × i / (n − 1))` (i = 정렬 인덱스, n = 모집단 유효 크기)
   - 가장 낮은 = RS 1, 가장 높은 = RS 99.

3. **모집단 외 종목 (코스닥·KOSPI 301위 이하 S 통과 종목)**
   - `rs = round(1 + 98 × lower / (n − 1))` (lower = 해당 수익률보다 낮은 모집단 종목 수)
   - 1~99 범위로 클램프.

### 4-3. RS 점수의 직관적 의미

| RS | 의미 |
|---|---|
| **99** | 상위 1% — KOSPI 시총 상위 300종목 중 가장 잘 오른 종목 |
| **80** | 상위 20% — *책 원전의 매수 권장선* (단, 우리 시스템에선 컷오프 아님) |
| **50** | 정확히 중앙값 — 시장의 절반보다 잘 오름 |
| **1** | 가장 못 오른 종목 |

## 5. 등급 라벨 (점수 분류 정보)

점수는 1~99 연속이지만, *책 원전의 RS 구간* 을 라벨로 함께 노출합니다. **라벨은 정렬·필터 보조 정보일 뿐 점수에 영향 없음.**

| RS | 라벨 | 책 기준 해설 |
|---|---|---|
| **95 ~ 99** | **최강 주도주** | 매수 우선 검토 |
| **90 ~ 94** | **강한 주도주** | 매수 가능 |
| **80 ~ 89** | **주도주** | 책의 매수 권장선 |
| **70 ~ 79** | **회색지대** | *책: "시장 수익률 뛰어난 주식 아님"* |
| **40 ~ 69** | **보통·소외주 동조** | *책: "절대 매수 금지"* |
| **1 ~ 39** | **소외주** | 매수 금지 |
| *데이터 없음* | — | 200일 미만 또는 차트 수집 실패 |

> 우리 시스템은 RS 1 종목도 *페이지에 그대로 노출* 합니다. 책의 매수 권고는 RS 80+ 지만, 그 아래 종목도 *점수로* 함께 볼 수 있어야 다른 페이지(C/A/N/S/I) 점수와 *교차 검토* 가 가능합니다.

## 6. RS 점수만 보고 매수하지 마라 — 진입 3대 조건

L 점수가 높다는 것은 *매수 후보 자격* 일 뿐 *매수 신호* 가 아닙니다. 페이지에서 *경고 박스* 로 강조하는 진입 조건 세 가지:

### 6-1. 모양 형성 확인 (책 #5)

차트가 *적절한 기간 동안 확실한 차트 모양(베이스)* 을 만들어냈는지 확인 후 매수.

- 컵-앤-핸들, 평행 베이스 등.
- *이 페이지는 모양을 자동으로 판정하지 않습니다.* 사용자가 일봉·주봉 차트를 직접 봐야 함.

### 6-2. 분기점(pivot) 매수 (책 #5)

정확한 매수 지점에서만 진입. 최초 매수 지점에서 **+5~10% 이상 상승한 다음에는 매수 금지** (추격 매수 회피).

> 오닐: 모양 형성 후 *돌파 직후 + 거래량 동반* 이 적기. 이미 5~10% 오른 종목은 *후속 조정 시 손절 위험* 이 커집니다.

### 6-3. -8% 손절 (책 #6)

> *"매수 가격보다 8% 떨어진 소외주가 있다면 당장 팔아버려라. 그렇게 하지 않으면 치명적인 상처를 입을 수 있다."*

CAN SLIM 공통 손절선이지만 L 원칙에서 특히 강조. RS 99 종목을 매수한 뒤에도 *예외 없이* 적용.

## 7. 정렬과 표시

### 기본 정렬

- **1차 키**: L 점수(RS) 내림차순. RS 99 → RS 1 순.
- **2차 키 (동점 시)**: *A 점수* 가 높은 종목 우선. *상대 강도 + 실적 진정성* 둘 다 강한 종목이 위로.
- **3차 키**: 종목 코드 사전순.
- **데이터 없음 (`rs_score = null`)** — 항상 가장 마지막 섹션.

### 필터·토글

- **시장**: KOSPI / 코스닥
- **모집단 내/외**: 코스닥·KOSPI 301위 이하 종목 필터
- **등급 라벨**: 최강·강한·주도주·회색지대·소외주 동조·소외주 칩

### 등급 라벨 (참고용, 점수에 영향 없음)

5절의 RS 구간 라벨이 그대로 사용됩니다. 메인 테이블에는 *raw 점수 + 라벨* 함께 노출 (예: `RS 87 (주도주)`).

### 메인 테이블 컬럼

| 컬럼 | 의미 |
|---|---|
| 종목명 + 코드 | |
| **L 점수** (RS) | 1~99 + 등급 라벨 (색 구분) |
| **52주 수익률** | RS 의 원본 수치 (참고용) |
| **A 점수** | 동점 정렬용 + 매수 우선순위 판단 |
| **현재가** | |
| **모집단** | "내" / "외 (추정)" 라벨 |

## 8. 자동 배지 — 핵심 3개만

배지는 *필터가 아니라 라벨* 입니다. L 점수가 핵심이고, 배지는 종목의 결을 한눈에 보여주는 보조 정보.

| 배지 | 조건 | 의미 |
|---|---|---|
| 🥇 최강 주도주 | RS ≥ 95 | KOSPI 300 중 상위 5% 수익률 |
| 🎯 RS+A 동시강함 | RS ≥ 80 AND A 점수 ≥ 40 | *가장 강한 매수 후보 라벨* |
| 📊 모집단 외 | `in_universe = false` | 코스닥 또는 KOSPI 301위 이하 — RS 는 *추정치* |

> **제거된 배지** — "강한 주도주", "회색지대", "소외주" 라벨. 이들은 *등급 라벨* 로 이미 표현되므로 배지 중복.

## 9. 자주 묻는 의문

### Q1. RS 30 인 종목이 왜 L 페이지에 보이나요? 책에선 매수 금지 구간 아닌가요?

**점수로서는 보입니다, 매수 후보로서는 아닙니다.**

이전 버전(v2)은 RS 80 미만을 별도 섹션으로 격리했지만, v3 부터 *컷오프 없는 연속 점수* 로 통합했습니다. 이유:

- 다른 페이지(C/A/N/S/I) 점수와 *교차 검토* 시 L 점수를 *부분 점수* 로 활용할 수 있어야 함.
- RS 30 종목이 C·A 가 매우 강하면 *모니터링 가치는 있음* — 시장이 아직 발견 못한 잠재 후보일 수 있음.
- 책의 "매수 금지" 권고는 *등급 라벨* 로 명확히 표시되므로 사용자가 오해할 위험 없음.

매수 시점은 RS 가 *자연스럽게 80+ 로 올라온 뒤*. 그 전까지는 *낮은 L 점수 = 보조 정보* 로 활용.

### Q2. 왜 KOSPI 만 모집단인가요? 코스닥도 포함해야 하지 않나요?

코스닥은 변동성·시총 분포·자금 흐름이 KOSPI 와 달라, 한 모집단에 섞으면 *작은 코스닥 종목이 큰 KOSPI 종목과 동일 척도* 로 비교됩니다. 그러면 코스닥 급등주가 RS 99 를 차지하고 KOSPI 진정한 주도주가 묻힙니다.

해결책: KOSPI 시총 상위 300 을 *모집단으로 고정*. 코스닥 종목은 *모집단 외* 표시 + RS 추정치 부여 (참고용).

### Q3. 모집단 외 (코스닥) 종목의 RS 추정치를 그대로 믿어도 되나요?

**보고용으로만 참고**. KOSPI 300 모집단의 수익률 분포로 *백분위 추정* 한 값이라, 코스닥 자체의 RS 분포와 다를 수 있습니다.

특히 코스닥은 변동성이 KOSPI 의 1.5~2배라 *같은 수익률이라도 코스닥에선 상대적으로 평범* 할 수 있습니다. 페이지에는 *모집단 외* 라벨이 함께 노출되므로 *정확한 비교는 어렵다* 는 점을 인식.

### Q4. RS 점수가 *반등 직전* 종목엔 잘 안 맞지 않나요?

**맞습니다.** RS 는 *지난 52주 누적 수익률* 의 백분위이므로:

- 지난 1년 *완만히* 오른 종목 → RS 높음.
- 지난 1년 *바닥에서 최근 1개월만 급등* 한 종목 → RS 낮음 (수익률 누적이 부족).

따라서 *바닥에서 막 돌아선 종목* 은 RS 점수가 낮게 잡힙니다. 오닐 책은 이 한계를 인정하면서도 *"확실한 모양 형성 + 신고가 돌파"* 가 동반되어야 RS 가 자연스럽게 80+ 로 올라간다고 봅니다.

→ **반등 초기 종목은 L 점수가 자연히 올라오기를 기다리세요.** 이 시기에는 C·A·N 등 다른 점수가 먼저 신호를 줍니다.

### Q5. RS 99 인데 모양도 안 만들었고 분기점도 없으면 매수 안 하는 게 맞나요?

**네, 매수 안 합니다.** L 점수 높음 = *상대 강도* 의 한 측면일 뿐, 모양·분기점·거래량 동반은 별도 확인 사항. RS 99 라도 차트가 이미 천정에서 하락 중이면 *추격 매수 금지* (책 #5 의 5~10% 룰).

### Q6. -8% 손절은 RS 가 높은 종목에도 무조건 적용되나요?

**예외 없이 적용.** RS 가 아무리 높아도 매수가 대비 -8% 면 즉시 매도. *상대 강도가 강했기 때문에 살아 돌아올 거라는 기대* 가 오히려 큰 손실로 이어집니다 (책 #6).

### Q7. L 점수 단독 비교가 어려운 두 종목 중 어느 쪽을 먼저 보나요?

같은 RS 점수라면 **A 점수가 높은 쪽** 우선. 페이지에 *A 점수* 컬럼이 함께 노출되는 이유. *상대 강도 + 실적 진정성* 동시 충족 종목이 가장 강한 후보.

자동 배지 🎯 **RS+A 동시강함** (RS ≥ 80 AND A ≥ 40) 이 붙은 종목은 *가장 강한 매수 후보 라벨* 로 따로 노출됩니다.

### Q8. L 점수가 100점 만점이라면 A 50점과 비교할 때 가중치가 두 배로 들어가는 거 아닌가요?

**페이지 자체에서는 가중합을 하지 않습니다.** L 페이지는 *L 점수만* 으로 정렬, A 페이지는 *A 점수만* 으로 정렬. 페이지마다 *자기 척도가 곧 정렬 키* 입니다.

페이지 *간* 종합 점수가 필요한 경우 (예: 종합 매수 후보 페이지) 는 각 페이지 점수를 *정규화* 해서 합산합니다. L 99 = A 50 = 정규화 1.0 같은 식. 이건 L 페이지 책임 밖.

### Q9. L 하나만 보면 충분한가요?

**부족합니다.** L 은 *시장의 상대적 평가* 만 봅니다.

실제 매수 결정은 다음 여섯을 *함께* 봐야 합니다:

- **C** — 현재 분기 실적
- **A** — 연간 실적의 진정성
- **N** — 카탈리스트 (신제품·신경영진·신고가)
- **S** — 수급 체질 (분할·부채)
- **L** — 시장 대비 강한가 ← *이 페이지*
- **M** — 시장 전체 추세
- **I** — 외인·기관 수급

여섯이 함께 충족돼야 비로소 매수 후보. RS 99 라도 시장(M)이 약세 전환 중이면 보류.

## 10. 데이터 갱신·운영 정보

- **선행 조건** — S 결과 (`can-slim-s-candidates.json`) 가 먼저 갱신되어 있어야 함.
- **실행** — `python scripts/fetch_l_rs.py` → `can-slim-l-candidates.json` 생성.
- **소요 시간** — Yahoo 1y 차트 호출 (모집단 300 + S 통과 종목 N개), 병렬 워커 20개로 **약 1~3분**.
- **핵심 로직** — `fetch_l_rs.py` 한 파일. S 통과 종목 코드를 입력 → Yahoo 차트 → 백분위 환산 → JSON 저장.
- **상수 설정** — `UNIVERSE_SIZE = 300`. **컷오프 상수(`RS_CUTOFF`)는 v3 부터 제거.**
- **데이터 부족 처리** — 일봉 < 200일이면 `rs_score = null`. 점수 미부여 = 평가 불가 (0점이 아님).
- **페이지 표시** — 상단 *생성일* 이 마지막 업데이트 시점. S 입력 종목 수, L 평가 종목 수, 모집단 외 종목 수, 데이터 부족 종목 수 함께 표시.

### 코드 영향 (v2 → v3 마이그레이션)

문서 v3 의 *컷오프 없는 점수 체계* 에 맞추기 위해 코드에서 다음을 정리해야 합니다:

- `RS_CUTOFF = 80` 상수 제거.
- `passes_l` 필드 제거 (또는 deprecated 표시 후 단계적 제거).
- `fail_reasons` 필드는 *데이터 부족 사유* 만 남기고, "RS < 컷오프" 사유는 제거.
- 결과 JSON 의 `excluded_count` → `data_missing_count` 로 의미 재정의.
- 페이지 컴포넌트에서 "통과 못한 종목" 섹션을 *점수 낮은 종목* 으로 통합, "데이터 없음" 만 별도 섹션 유지.
