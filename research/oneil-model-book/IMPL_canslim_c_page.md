# /stocks/canslim/c 페이지 — 어떤 종목이 보이는가

> "이 페이지에 뜬 종목은 무슨 기준으로 골라진 거지?" 한 문서로 답하기 위해.
> 2026-05-20 초안 박제. 2026-05-21 100점 스코어링 체계로 정리 (4축 구조).

## 한 줄 요약

**작년 같은 분기보다 이익이 크게 늘었고, 매출도 같이 늘었고, 그 성장 속도가 더 빨라지고 있는 종목** — 오닐의 'C' (Current Quarterly Earnings).

## EPS 가 뭐였더라

**EPS (주당순이익) = 기업의 세후 순이익 ÷ 현재 발행 보통주 수.**

오닐 'C' 가 보는 핵심은 EPS 의 **절댓값**이 아니라 **증감률** — 특히 *전년 동기 대비* 증가율과 그 증가율이 *직전 분기보다 더 가팔라졌는지*. 분기·연간 어닝 발표 때마다 우선 들여다봐야 하는 한 줄이 이 숫자.

---

## 페이지의 2단 구조

이 페이지는 **필터** 와 **점수** 가 분리되어 있습니다.

- **1단계 필터**: 5조건을 *전부* 통과한 종목만 테이블에 노출.
- **2단계 점수**: 통과 종목 사이의 *우선순위* 를 매기는 100점 만점 4축 스코어링.

---

## 1단계 — 페이지에 뜨려면 (5조건 동시 통과)

종목이 테이블에 노출되려면 아래 다섯 가지를 **전부** 만족해야 합니다.

1. **분기 EPS 가 작년 동기 대비 +25% 이상 늘었다.**
   - 작년 4분기 vs 올해 4분기처럼 **같은 분기끼리** 비교 (직전 분기 비교는 계절성 왜곡).
   - 적자→흑자 턴어라운드도 포함.
   - 이번 분기가 적자면 자동 탈락.
   - **+5~20% 같은 미미한 성장은 명시적으로 거부.** 오닐 권장 하한이 +25~30% 이고, 그 미만은 "C" 의 자격이 없다고 봅니다.

2. **매출도 같이 늘었다.** 둘 중 하나면 OK:
   - 분기 매출이 작년 동기 대비 +25% 이상, **또는**
   - 최근 3분기 매출 YoY 가 **엄격한 단조 증가**(a < b < c, 멈춤 없이 계속 커짐) AND 최신이 양수.
   - → "이익만 일회성으로 튄 종목"을 거르는 안전장치.

3. **성장 속도가 빨라지고 있다 (EPS 가속).**
   - 이번 분기 YoY 가 직전 분기 YoY 보다 더 높아야 함 (Δ > 0).

4. **2분기 연속 이익이 줄지 않았다.** (적신호 #1)

5. **성장률이 1/3 이하로 급감하지 않았다.** (예: +100% → +30% 같은 둔화 — 적신호 #2)

### EPS 가속 단계 — 색·아이콘으로 표시

가속의 *세기* 를 한눈에 보라고 3단계로 표시합니다. 가속 폭 Δ = 이번 YoY − 직전 YoY.

| 단계 | 아이콘 | 조건 |
|---|---|---|
| **가속** | ▲ | 0 < Δ ≤ 25%p |
| **강력 가속** | ▲▲ | 25 < Δ ≤ 100%p |
| **폭발 가속** | 🔥 | Δ > 100%p |

YoY 계산은 적자 종목도 동일 척도로 비교할 수 있게 *분모를 절댓값으로* 둡니다. **적자→흑자 턴어라운드도 자연스럽게 큰 Δ로 환산**되어 위 3단계 중 하나에 분류됩니다. 별도 "회복" 분기 없음.

> 오닐 책 핵심: *"성장률 절대값보다 직전 분기 대비 가속 폭이 가장 중요"*.

### 자동으로 붙는 배지 (필터 아님, 라벨일 뿐)

- **잠정** — 분기보고서가 아직 안 나왔고 잠정실적 기반.
- **⛔ 절대 매도 금지** — 매출·이익 *둘 다* 최근 3분기 연속 가속. 오닐 책 ("이런 종목은 들고 있어라").
- **🌱 슈퍼주니어** — 신규 상장(IPO 후 8~12분기 이내) 이면서 최근 분기 매출 YoY 가 **+100% 이상**을 한 번이라도 찍은 종목. 신생기업 중 폭발적으로 자라는 회사를 따로 표시.

## 2단계 — 100점 점수 체계 (3축)

5조건 필터를 통과한 종목들 사이에서 **누가 더 강한가** 를 매기는 점수. 정렬·기본 추천에 사용.

**총 100점 = ① 42점 + ② 38점(+α) + ③ 20점**

오닐 책의 무게중심을 반영해 **EPS 자체 평가(YoY 폭 + 가속) = 80점** 으로 5분의 4를 EPS 에 할당. 매출 검증이 나머지 20점.

> **경영진 품질은 C 가 아닌 A 원칙(ROE)에서 평가** (2026-05-21 변경). 이전 4축 체계의 경영진 7점은 ① +3, ② +3, ③ +1 로 분산.

축 ② 의 (+α) = 폭발 가속(Δ > 100%p)에서 100%p 초과분에 대한 추가 보너스. 38점 cap 을 초과 누적 가능 — 진짜 폭발적인 가속을 다른 폭발 종목과 구분하기 위해. 자세한 산식은 아래 축 ② 섹션.

### 축 ① — 분기 EPS YoY 폭 (42점)

오닐의 최소 권장 하한(+25~30%) 부터 점수 시작. 그 미만은 1단계에서 이미 걸러져서 등장하지 않음.

| 분기 EPS YoY | 점수 | 의미 |
|---|---|---|
| +25 ~ +30% | 18 | 오닐 권장 하한 통과 |
| +30 ~ +40% | 26 | 권장 |
| +40 ~ +70% | 33 | 우수 |
| +70 ~ +100% | 39 | 매우 우수 |
| +100% 이상 | 42 | 최고 (단, 강세장 막바지에서는 천정 신호일 수도 — 시장 추세는 *M 원칙*에서 다룸) |

### 축 ② — EPS 가속 폭 (38점 + 폭발 가속 초과 보너스)

오닐의 핵심 — "절대값보다 가속이 중요". 1단계의 ▲ / ▲▲ / 🔥 그대로 점수화.

| 가속 단계 | 조건 | 점수 |
|---|---|---|
| ▲ 가속 | 0 < Δ ≤ 25%p | 15 |
| ▲▲ 강력 가속 | 25 < Δ ≤ 100%p | 28 |
| 🔥 폭발 가속 | Δ > 100%p | 34 |

**3분기 연속 EPS 단조 증가 보너스: +4점** (base+bonus 합이 38 을 넘으면 38 로 cap).

**🔥 폭발 가속 초과 보너스: Δ 가 100%p 초과분에 대해 100 단위마다 +0.5점**. 38점 cap 을 넘어 누적 — 진짜 폭발적인 가속을 변별하기 위해 의도적으로 uncap (2026-05-21).

| Δ (가속 폭) | 초과 보너스 |
|---|---|
| 100 < Δ ≤ 200%p | +0 (폭발 가속 base 34점만) |
| 200 < Δ ≤ 300%p | +0.5 |
| 300 < Δ ≤ 400%p | +1.0 |
| 400 < Δ ≤ 500%p | +1.5 |
| Δ = N×100 ~ (N+1)×100%p | +0.5 × (N − 1) |

예: Δ = 1,000%p 면 폭발 34 + 3분기 보너스 4 (cap 38) + 초과 보너스 4.5 = **42.5점**. 축 ② 가 축 ① YoY 42점도 넘을 수 있음.

### 축 ③ — 매출 가속 (20점)

이익만 일회성으로 튄 게 아니라는 증거. 매출·이익 동행이 핵심.

| 조건 | 점수 |
|---|---|
| 분기 매출 YoY +25% 이상 | 11 |
| 최근 3분기 매출 YoY 엄격한 단조 증가 | +6 |
| 매출·이익 *둘 다* 3분기 연속 가속 (⛔ 매도금지 배지) | +3 |

총합 최대 20점 (캡).

### 종합 점수 → 등급

| 점수 합 | 등급 | 의미 |
|---|---|---|
| 80점 이상 | 🅐 강력 | 매수 후보 (4축 모두 강함) |
| 70 ~ 79 | 🅑 좋음 | 매수 검토 |
| 50 ~ 69 | 🅒 중립 | 워치리스트 |
| 50점 미만 | 🅓 약함 | 필터는 통과했지만 정렬 하위 |

워치리스트 페이지의 등급 기준과 동일 컷오프 — *한 종목의 점수를 다른 페이지와도 비교*할 수 있게 의도.

## 슈퍼주니어 — 신규 상장 모니터

오닐 책: *"신규 상장된 회사 중 최근 8~12분기에 걸쳐 매출 +100% 이상 증가율을 기록하는 회사가 있다면 무조건 모니터링 대상."*

### 슈퍼주니어 레이블 조건

- 상장 후 **8~12분기 이내** (IPO 후 약 2~3년 차).
- 상장 이후 *임의의 한 분기*에서 **분기 매출 YoY ≥ +100%** 를 기록.
- 직전 분기 EPS 가 흑자 또는 흑자 전환 임박.

### 노출 방식

- 5조건 필터를 통과한 종목 카드에 **🌱 슈퍼주니어** 배지 부착.
- 필터 미통과 종목 중 위 조건을 만족하는 종목은 페이지 하단 *"슈퍼주니어 모니터"* 패널에 별도 노출 (분모 부재로 5조건 통과가 구조적으로 어렵기 때문).

## 자주 묻는 의문

**Q. 작년에 적자였던 종목인데 어떻게 YoY % 가 나오죠?**
A. 적자→흑자 턴어라운드도 일반 종목과 같은 척도로 비교하기 위해 *분모를 절댓값으로* 둡니다. (분자: 이번 − 작년 / 분모: |작년|.) 큰 Δ로 환산되어 ▲ / ▲▲ / 🔥 중 하나로 분류됩니다.

**Q. YoY % 가 수천·수만 % 로 폭주하는 경우는요? (예: 32,165%)**
A. 작년 동기 EPS 가 0 에 가까우면 나눗셈이 폭주하는 수학적 노이즈입니다. 이를 억제하기 위해 **분모 하한 100원** 을 적용 — 작년 EPS 의 절댓값이 100원 미만이면 100원으로 간주해 계산. 예: 작년 EPS 가 ±3원 → 100원으로 보고 계산 → 32,165% 가 1,000%대로 정상화. 한국 EPS 의 의미 있는 최소 단위로 100원을 선택.

**Q. 이번 분기가 적자면요?**
A. C 원칙 자체가 부적격이라 페이지에 안 보입니다.

**Q. 회복(작년 적자 → 올해 흑자)은 별도 처리되나요?**
A. 아니요. 절댓값 분모 공식상 자연스럽게 큰 Δ 값이 나오므로 ▲ / ▲▲ / 🔥 중 하나에 흡수됩니다. 한국 기업은 1년 안에 폭발적으로 흑자전환되는 경우가 흔해, 이런 종목을 별도 카테고리로 빼지 않고 폭발 가속으로 그대로 노출시킵니다.

**Q. +100% YoY 종목은 무조건 좋은 거 아닌가요?**
A. **강세장 막바지에는 그게 천정 신호일 수 있습니다.** 이미 +100% 를 찍은 주도주는 주가가 이미 천정인 경우가 많습니다. 이 판단은 이 페이지가 아니라 *M (시장 추세) 원칙* 에서 다룹니다 — 시장이 약세 전환 신호를 보이면 C 페이지 상위 종목들의 신규 진입을 보류하세요.

**Q. C 하나만 보면 충분한가요?**
A. **부족합니다.** 이 페이지는 *오닐의 첫 글자(C)* 를 학습·참조하는 용도. 매수 결정은 L (상대강도), M (시장 추세), I (외인·기관 수급) 와 결합한 뒤에.

## 데이터 재생성 — 병렬 워커 방식 (2026-05-21 갱신)

코스피+코스닥 전체(약 2,600종목)를 단일 프로세스로 돌리면 2시간+. 워커 분할 + 영구 캐싱 + universe 사전필터로 10~30분에 끝낼 수 있다.

### 아키텍처

```
[FDR StockListing]  ──→  universe (KOSPI + KOSDAQ, 시총 포함)
   (KRX 공식, 안정)         │
                            ▼ 시총 미달 종목 사전 제외 (Naver 호출 X)
[Naver/Yahoo/DART]  ──→  Pass 1 (남은 종목만)
                            │
                            ▼ 캐시 hit 이면 통째 스킵
[.cache/canslim_stocks/]                                  ─→  worker_NN.json
                                                              │
                                                              ▼  reduce
                                                          can-slim-candidates.json
```

**핵심 개선** (2026-05-21):
- **종목 universe 와 시총을 FDR 로 일괄 조회** → Naver 차단 위험 큰 폭 감소. 시총 미달 종목(전체의 ~60%)은 Naver 를 한 번도 호출 안 함.
- **종목별 영구 캐싱** → 한 번 받은 종목은 `.cache/canslim_stocks/<코드>.json` 에 저장. 다음 실행 시 캐시 hit 이면 재요청 안 함. 부분 실패 후 재시도 안전·빠름.

### 1) 옛 캐시 청소 (선택, 보통 안 해도 됨)

```powershell
Remove-Item C:\Users\hanul\playground\my-stock\.cache\canslim_worker_*.json -ErrorAction SilentlyContinue
```

종목 캐시는 *지우지 마세요* — 24시간 TTL 이라 알아서 갱신됩니다. 완전 초기화하려면:
```powershell
python scripts/screen_canslim.py --clear-cache
```
또는
```powershell
Remove-Item C:\Users\hanul\playground\my-stock\.cache\canslim_stocks\*.json
```

### 2) 워커 N개 — 각각 새 PowerShell 창에서

```
C:\Users\hanul\playground\my-stock\scripts\canslim_worker.ps1 <인덱스> [<총워커수=10>]
```

예 (5개 워커, 각 창 시작 사이 30초~1분 텀 권장):
```
C:\Users\hanul\playground\my-stock\scripts\canslim_worker.ps1 0 5
C:\Users\hanul\playground\my-stock\scripts\canslim_worker.ps1 1 5
... (인덱스 0..4)
```

**캐시가 쌓인 후엔 재실행이 매우 빠릅니다** — 캐시 hit 비율이 90%+ 면 워커당 1~2분 안에 끝남.

### 3) 모두 끝나면 reduce 한 번

```
C:\Users\hanul\playground\my-stock\scripts\canslim_reduce.ps1
```

reduce 는 모든 워커 캐시 합치고 Pass 2(RS 백분위 계산) + `public/data/can-slim-candidates.json` 저장.

|---|---|---|
| `--no-cache` | off | 종목 캐시 비활성화. 모든 종목을 새로 fetch. |
| `--cache-ttl-hours N` | 24 | 종목 캐시 유효 시간. `0` = 영구(만료 없음). |
| `--clear-cache` | off | 시작 전에 종목 캐시 전체 삭제. |

### 워커 수 선택 가이드

- **너무 많이(10+)** = Naver/DART 서버 레이트 리밋에 걸려 후반부 워커 실패율 급증.
- **너무 적게(2~3)** = 워커당 시간이 길어 한 창이 30분+ 점유.
- 권장: **4~6개 워커, 각 창 사이 30초~1분 텀** 두고 시작.
- 캐시가 누적된 후엔 **워커 수 줄여도 빠름** (Naver 호출 자체가 적어짐).
