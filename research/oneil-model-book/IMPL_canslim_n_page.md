# /stocks/canslim/n 페이지 — 어떤 종목이 보이는가

> "이 페이지에 뜬 종목, 도대체 무슨 기준으로 골라진 거지?" 한 문서로 답하기 위해.
> 2026-05-21 v2. 신제품 단일 축 30점 스코어링으로 재정의.
> C 페이지(`IMPL_canslim_c_page.md`) 와 짝.

---

## 1. 한 줄 요약

> **"이번 분기 잘 벌었네"** 의 *이유* 가 — *경쟁 우위 있는 신제품* 이고, *이미 그 신제품으로 매출이 나고 있고*, *그 섹터에서 의미 있게 빛을 발하는* — 종목.

오닐의 CAN SLIM 에서 **N = New (신제품·신경영·신고가)** 입니다. 단, 이 페이지는 **신제품 하나만** 집중적으로 평가합니다.

| 페이지 | 보는 것 |
|---|---|
| **C** | 분기 실적이 잘 나왔는가 (모멘텀) |
| **N (여기)** | 그 실적의 *원인* 이 강한 신제품인가 |

### 신경영·신고가는 왜 빠졌나

오닐 책은 신제품·신경영·신고가 셋을 함께 다루지만:

- **신고가** — 차트·가격 데이터로 *기계적으로 확인 가능*. L 페이지의 RS · 추후 M 페이지의 시장 추세에서 더 정확하게 평가됨. N 에서 중복 평가할 필요가 없음.
- **신경영** — 한국 시장에서 경영진 교체가 *주가 상승 동인이 되는 케이스* 가 드물고, 대부분 사후적·해석적. 평가의 신호 대 잡음비가 낮음.
- **신제품** — *근본적이고 평가 가능한* 핵심 카탈리스트. 여기 집중.

> 따라서 N 페이지는 *신제품* 한 축만 봅니다. 신경영·신고가는 다른 글자에서 평가됩니다.

---

## 2. 입력 모집단 — C 통과 전부, 컷오프 없음

```
[C 5조건 필터 통과]  ─→  N 평가 (전부)
```

- C 페이지 *5조건 필터를 통과한 모든 종목* 이 N 페이지에 노출됩니다.
- **제외·컷오프 없음.** A 점수 컷, 매출 컷, RS 컷 등 *어떤 컷오프도 적용하지 않습니다.*
- C 통과 = N 노출. 노출된 종목의 *신제품 점수* 만 매겨 정렬.

> N 은 *판정* 이 아니라 *조사·정리·정렬* 의 단계입니다. C 통과 종목 중 어느 신제품이 더 강한지 사용자가 한눈에 비교할 수 있어야 하므로, 모집단을 미리 좁히지 않습니다.

## 3. 신제품 점수 — 30점 만점 (3축)

각 종목에 대해 *그 종목의 핵심 신제품 하나* 를 정해 세 축으로 평가합니다. 합 30점.

> **한 종목 = 한 신제품.** 회사가 여러 신제품을 동시에 출시 중이면 *주력 1개* 만 선택. 여러 개를 늘어놓으면 점수가 흐려집니다.

축별 무게중심:

| 축 | 만점 | 묻는 질문 |
|---|---:|---|
| ① 경쟁 우위 | **15** | 다른 기업 대비 우위에 놓여 있는가? |
| ② 현재 매출 기여 | **10** | 이미 매출이 크게 나고 있는가? |
| ③ 섹터·시장 임팩트 | **5** | 어떤 섹터에서 어떻게 빛을 발하고 있는가? |
| **합** | **30** | |

경쟁 우위에 가장 큰 가중치 15점 — *해자(moat) 가 있어야 신제품이 오래간다.* 매출이 일시적으로 크게 나도 경쟁사가 따라잡히면 신제품의 수명이 짧아지므로, *지속 가능한 우위* 가 N 의 핵심.

---

### 축 ① 경쟁 우위 (15점)

> *이 신제품이 다른 기업 대비 경쟁 우위에 놓여있는가?*

N 의 최대 가중치 축. *해자(moat)* 의 두께를 평가합니다.

| 점수 | 조건 |
|---:|---|
| **15** | 글로벌 또는 한국 시장에서 *유일·사실상 독점* — 대체재가 없거나 기술·특허·생산능력의 격차가 1~2년 이상 (예: HBM3E 12단 양산 가능 기업 한정) |
| **11** | 동급 경쟁사 2~5개 중 *명확한 우위 1~2위* — 기술 사양·고객사·납기 등에서 객관적 우위 확인 |
| **7** | 경쟁 가능 위치 — 동급 경쟁사와 *대등하거나 약간 앞섬*. 시장 점유율 유의미 |
| **4** | 경쟁사 추격 단계 — 후발이지만 기술적 차별점 일부 보유 |
| **0** | 경쟁 우위 확인 불가 또는 *me-too* — 다른 회사 제품과 본질적 차이 없음 |

> 채점 근거: 한국 언론사 기사 + 증권사 산업 리포트 + 회사 IR. **출처 URL 이 없으면 0점이 원칙.** 인상·짐작으로 점수 부여 금지.

---

### 축 ② 현재 매출 기여 (10점)

> *이 신제품으로 현재 매출이 크게 나고 있는가?*

가설이 아니라 *이미 발생한 매출* 인지가 핵심. *기대* 만의 신제품은 0점.

| 점수 | 조건 |
|---:|---|
| **10** | 전사 매출의 **30% 이상** 을 이 신제품이 차지 — *회사 정체성을 바꾼 제품* |
| **7** | 전사 매출 **15~30%** — 주력 제품으로 자리잡음 |
| **5** | 전사 매출 **5~15%** — 의미 있는 비중이지만 아직 주력은 아님 |
| **2** | 전사 매출 **< 5%** 이지만 *전년 동기 대비 매출 +100% 이상* 의 가파른 성장. 비중은 작지만 *기울기* 가 강함 |
| **0** | 매출 기여 미확인 또는 *상용화 전·시제품 단계* |

> 채점 근거: 회사 사업보고서·IR·분기 컨퍼런스콜의 *세그먼트별 / 제품별 매출*. 공시되지 않으면 0점. 사업보고서에 *공시된 세그먼트* 가 점수의 기준.

---

### 축 ③ 섹터·시장 임팩트 (5점)

> *어떤 섹터에서 어떻게 빛을 발하고 있는가?*

신제품 자체의 강도뿐 아니라 *그 신제품이 속한 섹터* 의 성장성·관심도까지 봅니다. 같은 신제품이라도 사양 섹터에 있으면 임팩트가 작고, 성장 섹터의 핵심 제품이면 임팩트가 큼.

| 점수 | 조건 |
|---:|---|
| **5** | *시장 주도 섹터의 핵심 제품* — 섹터 자체가 빠르게 성장 (CAGR 20%+) 중이고, 이 신제품이 그 섹터의 *대표 제품군* 으로 인식 (예: AI 가속기, 2차전지 양극재) |
| **4** | *성장 섹터의 보조 제품* — 섹터는 성장 중이지만 신제품이 핵심 위치는 아님 |
| **3** | *안정 섹터의 차별화 제품* — 섹터 성장률은 평범하지만 회사가 *그 안에서 두각* |
| **1** | *사양 섹터의 신제품* — 섹터가 축소 중. 회사 자체 성장은 가능해도 섹터 역풍 |
| **0** | 섹터 임팩트 확인 불가 |

> 채점 근거: 증권사 섹터 분석·산업 리포트·언론사 *섹터별 분류*. CAGR 등 수치 인용 가능하면 함께 코멘트.

## 4. 점수 → 등급

| 점수 | 등급 | 의미 |
|---|---|---|
| **24 ~ 30** | 🅐 **강력 신제품** | 세 축 모두 강함. 신제품 검증 완료. |
| **18 ~ 23** | 🅑 **검증 신제품** | 1~2축 강세, 1축 약점. 매수 검토 가능. |
| **10 ~ 17** | 🅒 **부분 신제품** | 신제품은 있으나 매출 기여 또는 경쟁 우위 약함. |
| **0 ~ 9** | 🅓 **미확인 / 약함** | 코멘트 미작성 또는 신제품 임팩트 불충분. |

C 페이지(80/70/50) · A 페이지(40/30/20)의 등급 컷오프와 동일한 *비율 구조* (80%·60%·33%·미달). 페이지를 넘나들며 *한 종목의 점수* 를 비슷한 척도로 읽을 수 있게.

## 5. 자동 계산 vs 수동 작성

신제품 평가는 *질적 판단* 이 핵심이라 자동화하지 않습니다. 사람이 한국 언론·산업 리포트·IR 을 직접 읽고 점수와 함께 *근거 코멘트* 를 남깁니다.

| 항목 | 자동 | 수동 |
|---|:---:|:---:|
| C 통과 종목 목록 동기화 | ✅ | |
| 핵심 신제품 선정 | | ✅ |
| 축 ① 경쟁 우위 점수 + 근거 | | ✅ |
| 축 ② 매출 기여 점수 + 근거 | | ✅ |
| 축 ③ 섹터 임팩트 점수 + 근거 | | ✅ |
| 출처 URL 수집 | | ✅ |
| 합산 점수 · 등급 산출 | ✅ | |

### JSON 스키마 (`n_commentary`)

```json
{
  "core_product": "HBM3E 12단",
  "scores": {
    "competitive_advantage": { "value": 15, "rationale": "글로벌 양산 가능 3개사 한정, 자사가 1위." },
    "revenue_contribution":  { "value": 7,  "rationale": "1Q26 메모리 사업부 매출의 22% 차지 (IR)." },
    "sector_impact":         { "value": 5,  "rationale": "AI 가속기 섹터의 핵심 부품, 시장 CAGR 35%." }
  },
  "total_score": 27,
  "tier": "A",
  "sources": [
    { "title": "...", "url": "..." }
  ],
  "researched_at": "2026-05-21"

```

코멘트 미작성 종목은 `n_commentary = null` 또는 `scores` 가 비어 있는 상태로 두면 됩니다. 점수는 자동으로 null·등급 🅓 미확인 처리.

## 6. 정렬과 표시

### 기본 정렬

- **1차 키**: 총 점수 내림차순.
- **2차 키 (동점 시)**: 축 ② 매출 기여 점수가 높은 종목 우선 (실현된 매출 > 가설).
- **3차 키**: 종목 코드.

### 컬럼 (테이블)

| 컬럼 | 내용 |
|---|---|
| 종목명 + 코드 | |
| 핵심 신제품 | core_product (한 줄) |
| ① 경쟁 우위 | 점수 (/15) |
| ② 매출 기여 | 점수 (/10) |
| ③ 섹터 임팩트 | 점수 (/5) |
| **N 점수** | 합 (/30) + 등급 배지 |

행 클릭 시 펼치면 각 축의 *근거 코멘트* 와 *출처 URL* 노출.

### 필터 토글 (선택)

- 등급별: 🅐 / 🅑 / 🅒 / 🅓 만 보기
- 미조사 종목 숨기기 (코멘트 미작성 종목 제외)

## 8. 자주 묻는 의문

### Q1. 왜 C 통과 전부인가요? 좁히지 않으면 조사 부담이 크지 않나요?

조사 부담이 크긴 합니다. 하지만 **컷오프로 좁히면 비교 정보가 사라집니다.** A 점수가 낮아 N 입력에서 빠진 종목 중에도 *신제품이 강한 종목* 이 있을 수 있고, 사용자가 *왜 이 종목이 빠졌는지* 확인할 수 없게 됩니다.

대신 **점수 → 등급 변환** 으로 사용자가 보고 싶은 등급만 필터로 추릴 수 있게 합니다 (🅐 강력만 보기 등). 조사 우선순위도 *C 점수 높은 종목부터* 자연스럽게 정해집니다.

### Q2. 신제품이 여러 개인 종목은 어떻게 점수 매기나요?

**가장 임팩트가 큰 신제품 1개** 만 선정합니다. 매출 기여, 시장 관심도, 회사 IR 의 핵심 메시지 기준. 여러 개를 평균 내거나 합산하지 않습니다 — *핵심 신제품 하나의 강도* 가 N 의 본질이기 때문.

여러 신제품이 모두 강하다면 그건 자연스럽게 *축 ② 매출 기여* 합계가 크게 잡혀 12점이 부여됩니다.

### Q3. 코멘트가 비어 있는 종목은 점수가 0인가요?

코멘트 미작성 종목은 **점수 null · 등급 🅓 미확인**. 0점과 구분됩니다 (0점은 조사했는데 임팩트가 없는 경우).

페이지 상단에 *조사 완료 / 미완료 비율* 이 표시되어 작업 진행도를 확인할 수 있습니다.

### Q4. 신제품이 *없는* 종목은요? 기존 제품 모멘텀으로 C 통과한 경우.

C 를 통과했는데 *신제품 카탈리스트가 없는 종목* 도 있습니다 (성숙 산업의 기존 제품 모멘텀 강화 등). 이 경우:

- 축 ① 경쟁 우위: 기존 제품 기준으로 0~7점 (신제품이 아니면 *상한 7* 로 채점)
- 축 ② 매출 기여: 신제품 매출이 없으니 0점
- 축 ③ 섹터 임팩트: 섹터 위치에 따라 0~3점

총점이 자연스럽게 낮아져 🅒 또는 🅓 등급으로 분류됩니다. *페이지에서 제외되지는 않습니다* — N 점수가 낮다는 *정보 자체* 가 의미가 있기 때문.

### Q5. 채점 기준이 주관적이지 않나요?

질적 평가라 주관성을 0 으로 만들 순 없습니다. 다만 세 가지 안전장치:

- **근거 코멘트 필수** — 각 축 점수에 *한 줄 이상의 객관적 근거* 를 남깁니다.
- **출처 URL 필수** — 한국 언론·산업 리포트의 URL 인용. 출처 없으면 0점.
- **0점이 디폴트** — 근거가 없으면 *높은 점수를 매기지 않는다* 가 원칙. 인상·짐작 금지.

이 세 가지로 채점자가 바뀌어도 *±2점 이내의 일관성* 을 목표.

### Q6. 축 ① 경쟁 우위가 15점으로 가장 큰 이유는?

신제품의 *수명* 을 결정하는 것은 *해자(moat)* 이기 때문입니다.

- 매출이 일시적으로 크게 나도 *경쟁사가 빠르게 따라잡히면* 신제품 수명이 짧고, 주가도 단기 모멘텀에 그칩니다.
- 반대로 *진입 장벽이 두꺼운* 신제품은 매출이 점진적으로 늘어도 *수년간 시장 지배력* 을 유지합니다.

따라서:

- **축 ① 경쟁 우위 (15점)** — *얼마나 오래 갈 신제품인가*. 최대 가중치.
- **축 ② 매출 기여 (10점)** — *지금 이미 매출이 나는가*. 실증 검증.
- **축 ③ 섹터 임팩트 (5점)** — *그 신제품이 빛나는 환경인가*. 보조 가중치.

매출이 *기대* 만 강한 종목보다 *해자가 두꺼운* 종목이 N 의 본질에 더 가깝다는 명시적 무게중심.

### Q7. N 점수가 높으면 무조건 매수인가요?

**아닙니다.** N 은 *신제품의 강도* 만 평가합니다. 매수 결정은:

- **C** — 분기 실적이 진짜인가
- **A** — 연간 실적의 진정성
- **N (여기)** — 그 실적의 원인이 강한 신제품인가
- **S** — 수급 체질이 우호적인가
- **L** — 시장 대비 상대강도가 주도주급인가
- **M** — 시장 추세가 강세인가
- **I** — 외인·기관 수급이 들어오고 있는가

여섯이 모두 충족되어야 비로소 매수 후보.

### Q8. 30점 만점은 어떻게 정해졌나요?

C 페이지 (100점) · A 페이지 (50점) 와의 *상대 비중* 입니다. 실적 검증(C+A=150점) 대비 신제품 검증의 비중을 *1/5 수준* 으로 설정 — 신제품이 중요하지만 *실적 없는 신제품 스토리* 만으로 매수하지 않겠다는 명시적 무게중심.

### Q9. 신고가 정보는 어디서 보나요?

L 페이지에서 52주 수익률·RS 점수로 평가합니다 (`IMPL_canslim_l_page.md`). N v2 에서는 신고가 자동 계산이 제거됐습니다.

## 9. 데이터 갱신·운영 정보

- **선행 조건** — C 결과 (`can-slim-candidates.json`) 가 먼저 갱신되어 있어야 함.
- **자동 부분** — C 통과 종목 목록을 N JSON 에 *동기화* (코드 없으면 추가, 떨어진 코드는 제거). 점수는 기존 코멘트 유지 + 신규 종목은 null 로 초기화.
- **수동 부분** — JSON 의 `n_commentary` 를 종목별로 작성. 핵심 신제품 · 세 축 점수 · 근거 · 출처.
- **코드 격리** — N 의 모든 로직은 N 파이프라인. C/A/S/L 코드는 수정하지 않음.
- **이전 버전 제거** — v1 의 `fetch_n_prices.py` (Yahoo 52주 신고가 계산) 는 v2 에서 사용하지 않음. JSON 스키마도 `current_price`/`high_52w`/`pct_from_52w_high` 필드 제거 예정.
- **페이지 표시** — 상단 *생성일* 이 마지막 입력 종목 갱신 시점. 종목별 *조사일* 은 `researched_at`.
