# /stocks/canslim/s 페이지 — 어떤 종목이 보이는가

> "이 페이지에 뜬 종목, 도대체 무슨 기준으로 골라진 거지?" 한 문서로 답하기 위해.
> 2026-05-21 v2. C/A/N 페이지 문서와 짝. (v1 의 컷오프·통과/미통과 이분법은 폐기, 점수제로 전환.)

---

## 1. 한 줄 요약

> **수요·공급 구조가 매수자에 우호적인 종목** — 주주환원(소각·배당) 가점, 주주가치 희석(증자·전환·스톡옵션) 감점, 부채비율 점수로 60점 만점 채점.

오닐의 CAN SLIM 에서 **S = Supply and Demand (수요와 공급)** 입니다.

- **C/A 페이지** — *이익이 진짜인가*
- **N 페이지** — *왜 그 이익이 나오는가 (카탈리스트)*
- **S 페이지 (여기)** — *주식 자체의 구조가 오를 수 있는가*

> 실적이 아무리 좋아도 *증자·전환 남발 + 차입금 폭증* 이면 주가는 둔하게 움직입니다. S 는 *주식 그 자체의 수급 체질* 을 점수로 환산합니다. 페이지에서 강제로 제외되는 컷오프는 *없습니다* — raw 점수와 함께 모든 종목을 나열, 사용자가 보고 판단.

---

## 2. 입력 모집단 — C 통과 종목 전체

```
[C 통과 종목 전체]  ─→  [S 평가 (60점 채점)]
```

- **C 페이지 통과 종목 = S 페이지 입력.** (N 통과 여부와 무관 — N 은 카탈리스트 조사용이고, 수급 체질은 *이익 검증을 통과한 모든 종목* 에 대해 봐야 함.)
- S 자체에서 DART · Naver 를 다시 호출해 자사주 소각·배당·증자 이력·부채비율을 *자력으로* 수집합니다. (C/A/N 결과는 종목 식별과 시총·가격 lookup 용도로만 재사용.)
- 데이터 수집 방식은 `/stocks/growth` 페이지가 사용하는 **shareholder-returns 파이프라인을 그대로 차용** — 점수 체계만 S 페이지 전용으로 재정의.

---

## 3. 점수 체계 — 총 60점

```
주주가치 (50점)  +  부채비율 (10점)  =  60점
```

강제 컷오프 없음. 모든 C 통과 종목이 점수와 함께 나열됩니다.

### 3-1. 주주가치 (50점) — 기본 25점에서 가감

기본값 **25점** 에서 출발해, *주주환원* 시마다 가점, *지분 희석* 시마다 감점. 결과는 `clamp(0, 50)`.

**가점 — 주주환원 행위**

| 항목 | 데이터 출처 | 가점 |
|---|---|---|
| 자사주 소각 (연도 수) | DART `tesstkAcqsDsposSttus` → growth 파이프라인 `treasury_cancellation_years` | 연 1회 = +5점 · 2년 = +10점 · 3년 이상 = +15점 |
| 연속 배당 (연도 수) | DART 배당 공시 → `consecutive_dividend_years` | 1년 = +2점 · 2~3년 = +5점 · 4년 이상 = +10점 |

**감점 — 주주가치 희석 이벤트** (최근 5년)

| 이벤트 유형 (DART `capital_changes`) | 감점 |
|---|---|
| 전환권행사 · 신주인수권행사 · 유상증자(제3자배정) · 주식매수선택권행사 · 상환권행사 | 건당 **−5점** |
| 유상증자(일반공모) · 유상증자(주주우선공모) · 유상증자(주주배정) | 감점 제외 (합리적 자금조달) |

> 희석 분류는 growth 페이지의 `DILUTIVE_TYPES` 집합을 그대로 사용 (기존 주주를 *일방적으로* 희석시키는 행위만 감점).

**최종**: `주주가치_점수 = clamp(25 + 가점 합 − 감점 합, 0, 50)`

> 가중치 (`+5`, `+2`, `−5` 등) 는 C 통과 종목 분포를 보고 조정될 수 있는 *튜닝 파라미터*. v2 출시 시점 초안값.

### 3-2. 부채비율 (10점)

**일반 산업** — 최근 분기 부채비율 (Naver 재무표 "부채비율" 행):

| 부채비율 | 점수 |
|---|---|
| ≤ 50% | **10점** |
| ≤ 100% | 7점 |
| ≤ 150% | 4점 |
| ≤ 200% | 2점 |
| > 200% | 0점 |

> 오닐: *"부채비율은 낮을수록 더 안전하고 더 나은 기업이다."* 부채 적은 기업은 금리 상승기에 EPS 가 안정.

**금융업 (KSIC 64·65·66)** — **5점 고정**.

| KSIC | 산업 |
|---|---|
| **64** | 금융업 (은행·신탁·자산운용) |
| **65** | 보험 및 연금업 |
| **66** | 금융 및 보험 관련 서비스업 (증권·선물중개) |

> 이유: 고객 예수금·RP 등이 부채로 잡혀 800~1,200% 가 *정상*. 일반 산업의 부채비율 점수표를 그대로 적용하면 모든 금융기관이 0점이 되는 왜곡 발생. 그렇다고 만점을 주면 *수급 평가의 의미* 가 사라지므로 **중간값 5점 고정** — "이 축은 평가 불가" 의 의미로 사용.

## 4. 정보 표시 (raw — 점수 외 참고)

점수와 별개로 카드에 표시되는 보조 정보. 점수에 영향 X, 사용자가 직접 *수급이 좋은가* 종합 판단하라는 의미.

### 4-1. 유통물량 / 전체 주식수 비율 (추계)

- **계산식**: `1 − (대주주·특수관계인 5%룰 합산 보유율 / 100)`
- **데이터 출처**: DART `majorstock` (5%룰 대량보유 보고).
- **의미**: *유통 가능* 한 주식 비율 추계. 작을수록 가벼움.

> 오닐: *"노련한 투자자는 발행수보다 유통 물량에 주목."* 동일 조건이면 유통이 적은 종목이 *의미 있는 매수 한 번에* 크게 움직임. (현재는 점수 미반영 — 분포 본 뒤 도입 검토.)

### 4-2. 최고 경영진 보유 % (5%룰 대량보유 보고자 합산)

- **데이터 출처**: 위와 동일 DART majorstock.
- **의미**: *경영진이 자기 회사 주식에 얼마나 애착이 있는가* 의 척도. 책 기준선: 대기업 1~3%, 중소기업은 그 이상.

### 4-3. 발행주식수

- **데이터 출처**: DART 기업개황.
- **의미**: 책 #1 "작을수록 가볍다". 점수는 부여하지 않음 (50억주여도 다른 축이 강하면 통과).

### 4-4. 부채비율 시계열 (현재 점수 + 추세)

- **수집 범위**: 최근 *5분기* + 최근 *3~5년 연간*.
- **표시 방식**:
  - 색: ≤ 50% 초록 / ≤ 100% 연두 / ≤ 150% 회색 / > 150% 빨강.
  - 추세 화살표: 가장 오래된 vs 최신 비교. 감소 ↓ (좋음, 초록), 증가 ↑ (나쁨, 빨강).

> 오닐: *"최근 2~3년 부채 상환한 기업은 이자비용 감소만으로 EPS 증가."* 추세 자체는 점수 미반영 (현재값만 부채비율 10점 산정에 사용) — 라벨로 보조 표시.

### 4-5. 주식 분할 이력 (최근 5년)

- **데이터 출처**: DART 분할 공시.
- **의미**: 책 #3 "두세 차례 분할은 천장 신호". 라벨로만 표시, 컷오프 폐기 (v1 의 "분할 3회 이상 제외" 규칙 삭제).

## 5. 라벨 (정보용 — 점수 외)

수치를 *해석한* 라벨. 점수에 영향 없지만 사용자가 *눈에 띄는* 종목을 빠르게 인식할 수 있게 부착.

| 라벨 | 조건 | 의미 |
|---|---|---|
| **소각** (초록) | `treasury_cancellation_years ≥ 2` | 2년 이상 자사주 소각 실적 |
| **배당** (노랑) | `consecutive_dividend_years ≥ 3` | 3년 이상 연속 배당 |
| **희석주의** (빨강) | `dilutive_event_count ≥ 3` | 5년 내 희석 이벤트 3건 이상 |
| **자사주 매우 큰 매입** | 자사주 보유율 ≥ 10% | 오닐 책 인용 |
| **연간 부채 크게 감소** | 최근 2~3년 연간 부채비율을 20%p 이상 낮춤 | 이자비용 감소로 EPS 자연 증가 |
| **분기 부채 크게 감소** | 최근 5분기 분기 부채비율을 20%p 이상 낮춤 | 단기 재무 개선 |
| **주식 분할 주의** | 최근 5년 분할 1회 이상 | 천장 신호 가능성 |
| **금융기관** | KSIC 64·65·66 | 부채비율 점수 5점 고정 안내 |

> 라벨 색상·아이콘은 growth 페이지의 `ShareholderBadges` 컴포넌트를 그대로 재사용.

## 6. 점수 정렬 — 통과/미통과 폐지

v1 의 *통과·미통과 이분법은 제거되었습니다*. 모든 C 통과 종목이 60점 만점 점수와 함께 한 테이블에 나열됩니다.

- **정렬** — S 점수 내림차순.
- **표시** — 각 카드에 `주주가치 / 부채비율` 세부 점수 + 라벨 + raw 보조 정보.
- **별도 컷오프 없음** — 사용자가 점수 분포를 직접 보고 매수 후보를 선별.

> "이 종목은 왜 보이지?" 가 아니라 "이 종목은 왜 *그 점수* 지?" 가 페이지의 질문이 됩니다. ScoreDetails 패널에서 가점·감점 내역이 모두 노출.

## 7. 오닐 'S' 8가지 핵심 (페이지 하단 학습 섹션과 동일)

| # | 핵심 | S 점수 반영 여부 |
|---|---|---|
| 1 | **발행 주식수 — 작을수록 가볍다.** 50억주는 둔하고 500만주는 가볍다. | raw 표시만, 점수 X |
| 2 | **유통 물량 / 경영진 지분.** 대기업 1~3%, 중소기업은 그 이상이 긍정 신호. | raw 표시만, 점수 X |
| 3 | **주식 분할 — 과도하면 천장 신호.** *"두세 차례 분할은 천장을 쳤다는 징후."* | 라벨만, 점수 X (v1 컷오프 폐기) |
| 4 | **자사주 매수·소각 — 강력한 긍정 신호.** *"10% 만 매수해도 무척 큰 것."* | **주주가치 가점 (+5~+15)** |
| 5 | **부채 비율 — 낮을수록 좋다.** 금리 상승 시 고부채 기업 EPS 급격 축소. | **부채비율 10점** |
| 6 | **거래량 — 수요·공급의 최고 척도.** 피벗 돌파 시 +40~50% 이상. | 페이지 자동 평가 X (사용자가 차트로 직접 확인) |
| 7 | **차트 모양 — 매주 관찰.** 주간 등락폭·거래량·종가 위치. | 페이지 자동 평가 X |
| 8 | **종합 — 자사주 + 경영진 지분 우선.** | 자사주 가점에 반영, 경영진 지분은 raw |

> #6·#7 은 *페이지가 자동으로 평가하지 않습니다*. 사용자가 일봉·주봉 차트를 직접 봐야 합니다. S 페이지는 #4·#5 에 해당하는 *정량 가능한 수급 체질* 만 점수로 환산.

## 8. 자주 묻는 의문

### Q1. v1 의 "분할 3회 이상 제외" / "부채비율 컷오프 TBD" 는 어디로 갔나요?

**둘 다 폐기되었습니다.** v2 는 강제 제외 컷오프를 운영하지 않습니다 — 부채비율은 *점수* 로 (10점 만점), 분할은 *라벨* 로만 표시. 사용자가 점수와 라벨을 함께 보고 직접 판단하는 구조입니다.

### Q2. 자사주 매입 결정 공시는 있는데 가점이 안 붙어요.

가점은 *소각 실적* 기준입니다 (`treasury_cancellation_years` → +5~+15). 매입 결정 공시는 raw 카운트로만 표시 — 실제 소각으로 이어져야 가점.

> 자사주 보유율 ≥ 10% 자체는 점수 X, "**자사주 매우 큰 매입**" 라벨로만 표시 (정보용).

### Q3. 발행주식수가 50억주인 종목도 점수가 나오나요?

**나옵니다.** 발행주식수 자체에는 점수도 컷오프도 없습니다. raw 로만 표시. 책 #1 의 "작을수록 좋다" 는 *동일 조건 비교 시* 의 진술이며, 50억주 종목 (예: 삼성전자) 도 다른 축이 강하면 높은 S 점수를 받을 수 있습니다.

### Q4. 부채비율 추세 화살표는 점수에 반영되나요?

**아니요.** 점수는 *최근 분기 부채비율* 만 사용. 추세는 라벨 ("연간 부채 크게 감소" / "분기 부채 크게 감소") 로만 표시.

- delta = oldest − latest, delta > 0 → 감소 ↓ (좋음)
- 5분기 시계열 우선, 부족 시 연간 사용

### Q5. 금융기관 5점 고정은 너무 후하지 않나요?

5점은 *10점 만점의 50%* — 의도적으로 "이 축은 평가하지 않음" 의 의미입니다. 만점(10점)이면 평가받지 않은 금융주가 다른 산업 대비 자동 우위가 되고, 0점이면 산업 전체가 일률적으로 페널티를 받는 왜곡 발생. 중간값이 합리적 절충.

> 금융업 면제는 *부채비율 점수만* — 자사주·배당·희석은 동일 기준으로 평가됩니다.

### Q6. 5%룰 보고 누락된 종목은 어떻게 처리되나요?

DART majorstock 응답이 비어 있으면 `insider_pct = null`, `float_ratio_estimated = null`. 페이지에서는 "—" 로 표시. 주주가치 점수는 자사주·배당·희석 이력으로만 산정되므로 majorstock 누락이 점수에 영향 X.

### Q7. CB(전환사채) 영향은 평가되나요?

**전환권행사 이벤트만 감점에 반영** (`DILUTIVE_TYPES` 에 포함). 발행 잔량 자체는 v2 미평가 — DART 에서 CB 발행·전환 잔량을 종목별로 자동 수집하기엔 복잡도가 커서 v2 는 *실제 전환된 이벤트* 만 봅니다. 사용자가 CB 가 많은 종목인지 별도 확인 권장.

### Q8. S 점수 60점이 만점인데, 몇 점 이상이 "좋은" 건가요?

**분포가 쌓여야 답할 수 있습니다.** v2 출시 직후엔 절대 임계 없음 — 사용자가 C 통과 종목 전체의 S 점수 분포를 보고 상대 평가. 분포가 안정화되면 (수 회 운영 후) "55점 이상 → 매수 후보 풀" 같은 가이드를 별도로 정할 예정.

### Q9. S 하나만 보면 충분한가요?

**부족합니다.** S 는 *수급 체질* 만 봅니다. 매수 결정은 C/A (실적) · N (카탈리스트) · L (상대강도) · M (시장) · I (수급 흐름) 와 함께. 특히:

- *발행주식수가 작아 가볍다* → 상승만 빠른 게 아니라 *하락도 빠름*. M (시장 추세) 약세 신호 시 우선 점검.
- *S 점수 높음* → 매수 후 -8% 손절 적용은 동일. 신뢰가 손절 규칙을 면제하지 않음.

## 9. 데이터 갱신·운영 정보

- **선행 조건** — C 결과 (`can-slim-c-candidates.json`) 가 먼저 갱신되어 있어야 함. (N 결과는 불필요.)
- **shareholder-returns 파이프라인 재사용** — `/stocks/growth` 가 사용하는 `public/data/shareholder-returns.json` 을 *그대로* 입력으로 사용. C 통과 종목 중 누락된 종목만 `scripts/fetch-shareholder-returns.ts` 로 보충 수집.
- **실행** — `python scripts/screen_canslim_s.py` → `can-slim-s-candidates.json`. (CLI 옵션 폐기 — 컷오프가 없으므로 `--debt-threshold` / `--debt-reduction` 옵션도 폐기.)
- **소요 시간** — 종목당 DART 호출 4~7회 (majorstock · list · tesstkAcqsDsposSttus · capital_changes · 분할공시) + Naver 재무표 → C 입력 100~200종목 기준 **약 15~30분**. shareholder-returns 캐시가 있으면 단축.
- **코드 격리** — S 의 모든 로직은 `canslim_lib/criteria_s.py`. C/A/N 코드는 수정하지 않으며 결과 JSON 도 *읽기 전용 입력* 으로만 사용 (시총·가격·종목명은 C JSON 에서 lookup).
- **점수 함수** — `score_s(stock, shareholder_return, debt_ratio, is_financial) -> { score, shareholder_score, debt_score, details, badges }`. growth 페이지의 `scoreGrowth` Cat4 (주주환원 보정) 로직을 50점 스케일로 이식.
- **분포 출력** — 매 실행 끝에 C 통과 종목의 S 점수 분포 (min/25%/50%/75%/90%/max) 가 stdout 으로 출력 — 추후 가중치 튜닝용 보조 데이터.
- **페이지 표시** — 상단 *생성일* 이 마지막 업데이트 시점. 점수 내림차순 단일 테이블, 카드별 ScoreDetails 패널.
